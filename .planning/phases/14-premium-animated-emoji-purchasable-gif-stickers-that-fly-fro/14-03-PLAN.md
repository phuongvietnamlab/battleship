## Objective

Create the animated GIF/spritesheet assets for the 6 premium emoji, add full i18n support, wire up the avatar-targeted animation positioning, and handle edge cases (mobile responsiveness, screen receiver shake effect for "boxing", concurrent animations graceful handling).

### Task 2: Avatar-targeted positioning

**Files:** `public/app.jsx`, `public/style.css`

Refine the animation system to target actual avatar element positions:

```javascript
// Use refs or query selectors to find avatar positions
function getAvatarPositions() {
  const myAvatar = document.querySelector('.pc-left .pc-avatar');
  const oppAvatar = document.querySelector('.pc-right .pc-avatar');
  // Or use the player-card containers
  const myRect = myAvatar?.getBoundingClientRect();
  const oppRect = oppAvatar?.getBoundingClientRect();
  return { my: myRect, opp: oppRect };
}
```

Update `PremiumEmojiAnimation` to compute start/end positions dynamically based on actual avatar element positions rather than fixed percentages. Fall back to percentage-based if elements not found.

Responsive handling:
- On mobile (< 768px): avatars are typically stacked or in a different layout — detect and adjust animation path accordingly
- Ensure animation stays within viewport bounds

**Acceptance:**
- Projectile starts from sender's avatar and arrives at receiver's avatar
- Works correctly on both desktop (side-by-side layout) and mobile (stacked layout)
- Falls back gracefully if avatar elements not found in DOM

### Task 5: Edge cases + mobile polish

**Files:** `public/app.jsx`, `public/style.css`

Handle edge cases:

1. **Game ends during animation:** If `gameOver` becomes true while an animation is playing, let it finish (don't abruptly remove)
2. **Rapid reconnect:** On socket reconnect, clear any stale animation queue
3. **Multiple animations:** If multiple premiumEmoji events arrive within 2.2s, queue them with a 500ms stagger (don't overlap)
4. **Mobile tap:** Ensure emoji picker buttons have adequate touch targets (min 44×44px)
5. **Balance sync:** After sending an emoji, update the displayed balance in the picker immediately (optimistic from callback, confirmed via `balanceUpdate` event)
6. **Bot games:** Premium emoji disabled/hidden when playing vs bot (no opponent to receive)

```javascript
// Disable premium emoji in bot games
const showPremiumEmoji = !vsBot && room?.started && !room?.gameOver;
```

**Acceptance:**
- No crashes on game-end during animation
- No stale animations after reconnect
- Concurrent animations staggered visually
- Touch targets ≥ 44px on mobile
- Balance updates immediately in picker after send
- Premium emoji hidden/disabled in bot games
