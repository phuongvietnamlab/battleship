# Phase 14 — Plan 02: Frontend — Emoji Picker UI + Animation System

## Objective

Build the client-side premium emoji picker integrated into the ChatComposer, and a full-screen animation system that plays the fly-and-impact effect when a `premiumEmoji` socket event is received. Both players see the animation regardless of who sent it.

## Tasks

### Task 2: Premium emoji picker tab in ChatComposer

**Files:** `public/app.jsx`, `public/style.css`

Extend ChatComposer with a premium tab:

- Add a tab toggle between "Free emoji" (existing CHAT_EMOJIS) and "Premium ✨" (premium animated emoji)
- Premium tab shows a 3×2 grid of emoji cards, each showing:
  - Preview: emoji_char (large) or thumbnail
  - Cost badge: "5 pts" overlay
  - Disabled state (greyed + locked icon) if user balance < cost
  - Greyed entirely with "Đăng nhập / Sign in" text if user is guest
- When cooldown active: all premium emoji show a countdown timer overlay
- Show current balance at the top of premium tab

```jsx
function PremiumEmojiPicker({ emojis, balance, isGuest, cooldown, onSend, lang }) {
  if (isGuest) {
    return (
      <div className="premium-emoji-locked">
        <span>{lang === "vi" ? "Đăng nhập để dùng" : "Sign in to use"}</span>
      </div>
    );
  }
  return (
    <div className="premium-emoji-grid">
      <div className="premium-balance">{balance} pts</div>
      {emojis.map(e => (
        <button
          key={e.id}
          className={"premium-emoji-btn" + (balance < e.cost || cooldown ? " disabled" : "")}
          disabled={balance < e.cost || cooldown}
          onClick={() => onSend(e.id)}
          title={lang === "vi" ? e.description_vi : e.description_en}
        >
          <span className="pe-char">{e.emoji_char}</span>
          <span className="pe-cost">{e.cost}pts</span>
          {cooldown && <span className="pe-cooldown">⏳</span>}
        </button>
      ))}
    </div>
  );
}
```

**Acceptance:**
- Tab toggle visible in ChatComposer between free and premium emoji
- Premium grid shows 6 emoji with cost badges
- Disabled state works correctly for insufficient balance
- Guest user sees "sign in" prompt instead of emoji grid
- Cooldown indicator shows when active
- i18n: descriptions follow current language

### Task 3: Send premium emoji via socket + cooldown handling

**Files:** `public/app.jsx`

Handle the send action:

```javascript
function sendPremiumEmoji(emojiId) {
  if (emojiCooldown) return;
  socket.emit("sendPremiumEmoji", { emojiId }, (res) => {
    if (res && res.ok) {
      // Start 5s cooldown locally
      setEmojiCooldown(true);
      setTimeout(() => setEmojiCooldown(false), 5000);
      // Balance updated via balanceUpdate event
    } else if (res && res.code === "COOLDOWN") {
      // Already cooling down — do nothing (UI should prevent this)
    } else if (res && res.code === "INSUFFICIENT_BALANCE") {
      // Optional: flash balance red briefly
    }
  });
}
```

**Acceptance:**
- Clicking a premium emoji emits socket event with emojiId
- On success: 5s local cooldown starts, buttons disabled
- On error: no animation triggered, user sees appropriate state
- Cannot re-send while cooldown active

### Task 4: Animation overlay component — flight + impact

**Files:** `public/app.jsx`, `public/style.css`

Create a full-viewport animation overlay that renders the emoji flight path:

```jsx
function PremiumEmojiAnimation({ event, myClientId, onComplete }) {
  // event: { emojiId, slug, senderId, ts }
  // Determine direction: sender → receiver
  // If senderId === myClientId: fly from MY avatar → OPPONENT avatar (left → right)
  // Else: fly from OPPONENT avatar → MY avatar (right → left)

  const isFromMe = event.senderId === myClientId;
  const direction = isFromMe ? "ltr" : "rtl";

  useEffect(() => {
    // Auto-dismiss after animation completes (~2s)
    const timer = setTimeout(onComplete, 2200);
    return () => clearTimeout(timer);
  }, []);

  return (
    <div className={`premium-anim-overlay ${direction}`}>
      <div className={`premium-anim-projectile anim-${event.slug}`}>
        <img src={`/emojis/${event.slug}.gif`} alt="" />
      </div>
      <div className={`premium-anim-impact impact-${event.slug}`} />
    </div>
  );
}
```

### Task 5: Listen for `premiumEmoji` socket event and trigger animation

**Files:** `public/app.jsx`

Wire up the socket listener in the battle screen:

```javascript
const [emojiAnimQueue, setEmojiAnimQueue] = useState([]);

useEffect(() => {
  socket.on("premiumEmoji", (data) => {
    // data: { emojiId, slug, senderId, ts }
    setEmojiAnimQueue(prev => [...prev, { ...data, key: data.ts + "_" + data.slug }]);
  });
  return () => socket.off("premiumEmoji");
}, []);

function handleAnimComplete(key) {
  setEmojiAnimQueue(prev => prev.filter(e => e.key !== key));
}

// In render:
{emojiAnimQueue.map(ev => (
  <PremiumEmojiAnimation
    key={ev.key}
    event={ev}
    myClientId={clientId}
    onComplete={() => handleAnimComplete(ev.key)}
  />
))}
```

**Acceptance:**
- Both players see the animation when either sends a premium emoji
- Animation direction is correct relative to each player's perspective
- Animations queue and play sequentially if multiple arrive
- No memory leak (events cleaned up on unmount)
