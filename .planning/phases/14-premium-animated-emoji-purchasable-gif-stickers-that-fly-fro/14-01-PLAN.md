# Phase 14 — Plan 01: Backend — Database, API & Socket Handler

## Objective

Create the server-side infrastructure for premium animated emoji: database table for the emoji catalog, a public API endpoint to fetch available emoji, and a Socket.IO event handler that validates balance, deducts points, enforces cooldown, and broadcasts the animation event to the room.

### Task 2: DB helper functions in `db.js`

**Files:** `db.js`

Add functions:

```javascript
// Get all active premium emoji (for API/cache)
async function getPremiumEmojis() {
  const { rows } = await pool.query(
    "SELECT id, name, slug, emoji_char, cost, description_en, description_vi, animation_file, impact_type, sort_order FROM premium_emojis WHERE active = true ORDER BY sort_order"
  );
  return rows;
}

// Get single emoji by ID (for validation in socket handler)
async function getPremiumEmojiById(emojiId) {
  const { rows } = await pool.query(
    "SELECT id, slug, cost, active FROM premium_emojis WHERE id = $1",
    [emojiId]
  );
  return rows[0] || null;
}
```

Export both from `db.js`.

**Acceptance:**
- Functions callable and returning correct data
- Exported in module.exports

### Task 3: GET /api/emojis endpoint

**Files:** `server.js`

Add a public (no auth required) cached endpoint:

```javascript
// In-memory cache for emoji catalog (refresh every 5 minutes)
let emojiCache = null;
let emojiCacheAt = 0;
const EMOJI_CACHE_TTL = 300_000; // 5 minutes

app.get("/api/emojis", async (req, res) => {
  const now = Date.now();
  if (!emojiCache || now - emojiCacheAt > EMOJI_CACHE_TTL) {
    try {
      emojiCache = await getPremiumEmojis();
      emojiCacheAt = now;
    } catch (e) {
      console.error("[emojis] fetch failed:", e.message);
      return res.status(500).json({ error: "SERVER_ERROR" });
    }
  }
  res.json({ emojis: emojiCache });
});
```

**Acceptance:**
- `GET /api/emojis` returns `{ emojis: [...] }` with 6 items
- Subsequent requests within 5 min served from cache
- No auth required

### Task 4: Socket event `sendPremiumEmoji` with validation + debit

**Files:** `server.js`

Add rate limiter + socket handler:

```javascript
// Rate limiter: 1 premium emoji per 5 seconds per player
const premiumEmojiLimiter = new RateLimiterMemory({ points: 1, duration: 5 });

socket.on("sendPremiumEmoji", async (arg, cb) => {
  // Rate limit: 5s cooldown
  const rlKey = socket.data.clientId || socket.id;
  try {
    await premiumEmojiLimiter.consume(rlKey);
  } catch (e) {
    return cb && cb({ ok: false, code: "COOLDOWN" });
  }

  // Must be authenticated
  const userId = socket.data.userId;
  if (!userId) return cb && cb({ ok: false, code: "AUTH_REQUIRED" });

  // Must be in a room in battle phase
  const code = socket.data.roomCode;
  const clientId = socket.data.clientId;
  const room = rooms[code];
  if (!room || !room.started || room.gameOver) {
    return cb && cb({ ok: false, code: "NOT_IN_BATTLE" });
  }

  // Validate emoji exists and is active
  const emojiId = arg && arg.emojiId;
  if (!emojiId || typeof emojiId !== "number") {
    return cb && cb({ ok: false, code: "INVALID_EMOJI" });
  }

  const emoji = await getPremiumEmojiById(emojiId);
  if (!emoji || !emoji.active) {
    return cb && cb({ ok: false, code: "INVALID_EMOJI" });
  }

  // Debit wallet atomically
  const referenceId = `emoji_${emoji.slug}_${Date.now()}_${socket.id}`;
  const debitResult = await debitWallet(userId, emoji.cost, "emoji_purchase", referenceId);
  if (!debitResult.ok) {
    return cb && cb({ ok: false, code: "INSUFFICIENT_BALANCE" });
  }

  // Broadcast animation to both players
  const opp = opponentOf(room, clientId);
  const payload = {
    emojiId: emoji.id,
    slug: emoji.slug,
    senderId: clientId,
    ts: Date.now()
  };
  // Send to both players so both see the animation
  emitToClient(room, clientId, "premiumEmoji", payload);
  if (opp) emitToClient(room, opp, "premiumEmoji", payload);

  // Push updated balance to sender
  socket.emit("balanceUpdate", { balance: debitResult.balance });

  touchRoom(room);
  cb && cb({ ok: true, balance: debitResult.balance });
});
```

**Acceptance:**
- Unauthenticated user gets `AUTH_REQUIRED`
- User not in battle gets `NOT_IN_BATTLE`
- Invalid/inactive emoji ID gets `INVALID_EMOJI`
- Insufficient balance gets `INSUFFICIENT_BALANCE`
- Within 5s cooldown gets `COOLDOWN`
- Successful send: points deducted, both players receive `premiumEmoji` event, sender gets `balanceUpdate`
- Transaction recorded in `transactions` table with type `emoji_purchase`
