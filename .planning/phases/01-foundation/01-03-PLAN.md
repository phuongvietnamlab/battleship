<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client fire payload → doShot | Untrusted, possibly malformed, shot payload crosses into resolution logic |
| client → rooms map | Continuous/abandoned room creation crosses into long-lived in-memory state |
| client profile/chat → DB + other clients' DOM | User-supplied strings stored and later rendered (stored XSS surface) |
| browser → HTTP responses | CSP governs what the page may load/execute |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02 | Denial of Service | doShot resolution | mitigate | Guard-clause null/shape + cells-array check returns BAD_STATE instead of throwing — a crash-probe cannot kill a game (SEC-02, CONCERNS #6). |
| T-03-03 | Denial of Service | rooms map memory | mitigate | Hybrid sweep evicts empty + idle rooms every 60s, bounding memory by active games (SEC-03, CONCERNS #8). |
| T-03-04 | Tampering/Info Disclosure (Stored XSS) | profile name + chat | mitigate | sanitizeProfile HTML-escapes + control-char-strips names before storage; sanitizeChat sanitizes chat; CSP `script-src 'self'` blocks inline-script execution as defense in depth (SEC-04, CONCERNS #3). |
| T-03-E1 | Elevation (script execution) | client page | mitigate | Content-Security-Policy header with `script-src 'self'`, `frame-ancestors 'none'`, no unsafe-inline in script-src. |
</threat_model>
