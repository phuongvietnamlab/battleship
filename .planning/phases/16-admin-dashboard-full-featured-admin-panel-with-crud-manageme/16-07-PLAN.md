# Plan 07: Admin frontend scaffold

## Objective
Build the admin SPA scaffold — HTML shell, React entry point with hash-based routing, sidebar layout, theme system, reusable DataTable component, chart wrapper, toast/confirm UI, loading states, API hooks, WebSocket hook, i18n translations, and esbuild integration.

## must_haves
- [ ] Admin SPA builds independently via extended build-game.mjs
- [ ] Hash-based routing navigates between admin pages without React Router
- [ ] Collapsible sidebar with nav links and active state
- [ ] Dark/light theme toggle persisted in localStorage
- [ ] Reusable DataTable with pagination, sorting, row selection, bulk actions
- [ ] Recharts wrapper for analytics charts
- [ ] Toast notification system and confirmation dialog
- [ ] API fetch hook with admin session handling
- [ ] Socket.IO hook for real-time stats
- [ ] Full i18n (EN/VI) for admin UI strings

### Task 2: React app entry with hash routing and layout

<read_first>
- public/admin/app.jsx
- public/app.jsx
</read_first>

<action>
Create public/admin/app.jsx as the main admin SPA entry:

1. Import React and ReactDOM (same as game app.jsx pattern — bundled via esbuild)
2. Implement useHashRoute() hook:
   - State: route initialized from window.location.hash.slice(1) || "/dashboard"
   - Effect: listen to "hashchange" event, update route
   - Return: current route string

3. Implement App component:
   - Call useHashRoute() to get current route
   - If route === "/login", render Login page
   - Otherwise render Layout component wrapping the appropriate page based on route:
     "/dashboard" → Dashboard, "/users" → Users, "/matches" → Matches,
     "/content" → Content, "/moderation" → Moderation, "/operations" → Operations
   - Default/unknown routes: redirect to "/dashboard"

4. Implement simple auth state:
   - useState for adminUser (null until logged in)
   - useEffect on mount: call GET /api/admin/me. If 200, set adminUser. If 401, navigate to #/login.
   - Pass adminUser + setAdminUser as props to Layout and pages

5. ReactDOM.createRoot(document.getElementById("admin-root")).render(<App />)

Create public/admin/components/Layout.jsx:
1. Accept props: children, adminUser, route
2. Render: sidebar (Sidebar component) + main content area
3. Main content area wraps {children} in a div with className "admin-content"
4. Add header bar with: admin user name display, theme toggle button, logout button

Create public/admin/components/Sidebar.jsx:
1. Accept props: route, adminUser
2. Render nav items as a list: Dashboard, Users, Matches, Content, Moderation, Operations
3. Each item is an anchor tag with href="#/pagename"
4. Active state: add "active" class when route matches the item's path
5. Collapsible: toggle state via a hamburger button at top; collapsed shows only icons
6. Role-based visibility: hide "Operations" for moderator role
7. Store collapsed state in localStorage
</action>

<acceptance_criteria>
- Admin app renders at /admin with React
- Hash routing works: navigating to #/users shows Users placeholder
- Unauthenticated access redirects to #/login
- Authenticated admin sees sidebar + content area
- Sidebar highlights the active route
- Sidebar can be collapsed/expanded and state persists in localStorage
- Logout button calls POST /api/admin/logout and redirects to #/login
</acceptance_criteria>

### Task 4: DataTable reusable component

<read_first>
- public/admin/components/DataTable.jsx
</read_first>

<action>
Create public/admin/components/DataTable.jsx accepting props:

Props interface:
- columns: [{ key, label, sortable, render }] — column definitions
- data: array of row objects
- total: total row count (for pagination)
- page: current page number
- limit: rows per page
- onPageChange(newPage): callback
- onSort(key, order): callback
- sortKey: current sort column
- sortOrder: "asc" or "desc"
- selectable: boolean (show checkboxes)
- onSelectionChange(selectedIds): callback
- bulkActions: [{ label, onClick, variant }] — toolbar buttons when rows selected
- loading: boolean
- emptyMessage: string shown when data is empty

Component behavior:
1. Render table with thead (sortable column headers with arrow indicators) and tbody (data rows)
2. If selectable, first column is a checkbox (header = select-all toggle, rows = individual select)
3. Pagination footer: show "Showing X-Y of Z" text, prev/next buttons, page number display
4. When loading=true, show Loading component overlay
5. When data is empty and not loading, show emptyMessage centered
6. Column render: if column has render function, call it with (value, row). Otherwise display raw value.
7. Bulk actions toolbar: appears above table when selectedIds.length > 0, shows action buttons
8. Apply .data-table CSS class for styling
</action>

<acceptance_criteria>
- DataTable renders column headers from columns prop
- Clicking sortable column header calls onSort with key and toggled order
- Sort indicator (arrow up/down) shows on active sort column
- Checkbox selection works: select-all toggles all visible rows
- Pagination shows correct range text and disables prev/next at boundaries
- Loading state shows skeleton overlay
- Empty state shows custom message
- Bulk action toolbar appears when rows are selected
- Component is generic — no domain-specific logic
</acceptance_criteria>

<action>
Create public/admin/components/Chart.jsx:
1. Import LineChart, BarChart, Line, Bar, XAxis, YAxis, CartesianGrid, Tooltip, Legend, ResponsiveContainer from recharts
2. Export LineChartWrapper({ data, xKey, lines, height }) — renders ResponsiveContainer > LineChart with configurable lines array [{ dataKey, color, name }]
3. Export BarChartWrapper({ data, xKey, bars, height }) — renders ResponsiveContainer > BarChart with bars array [{ dataKey, color, name }]
4. Both use theme-aware colors: text fill from CSS variable via getComputedStyle, grid stroke from --border

Create public/admin/components/Toast.jsx:
1. Toast context provider: ToastProvider wraps children, manages array of toasts
2. Each toast: { id, message, type (success/error/warning/info), duration }
3. useToast() hook returns { showToast(message, type, duration) }
4. Toast container: fixed position top-right, renders toast items with auto-dismiss (default 4 seconds)
5. Toast item: colored left border based on type, text, close button
6. Animation: slide-in from right, fade-out on dismiss

Create public/admin/components/Confirm.jsx:
1. ConfirmProvider wraps children, manages confirm dialog state
2. useConfirm() hook returns { confirm(options) } where options = { title, message, confirmText, type }
3. Returns a Promise that resolves true (confirmed) or false (cancelled)
4. For destructive actions (type="danger"): show a text input where user must type the confirmText to enable the confirm button
5. Dialog: modal overlay with card, title, message, cancel + confirm buttons
6. Confirm button disabled until typed text matches (for danger type)

Create public/admin/components/Loading.jsx:
1. Skeleton loader component: renders animated placeholder bars (pulse animation)
2. Props: lines (number of skeleton lines to show, default 5), type ("table" | "card" | "text")
3. For "table": render skeleton rows with alternating widths
4. For "card": render card-shaped skeleton blocks
5. CSS animation: @keyframes pulse with opacity shift
</action>

- BarChartWrapper renders a Recharts bar chart with responsive container
- Both chart components adapt to theme (read CSS variable colors)

### Task 6: API hook, Socket hook, and i18n

<read_first>
- public/admin/hooks/useApi.jsx
- public/admin/hooks/useSocket.jsx
- public/admin/i18n/admin.js
</read_first>

<action>
Create public/admin/hooks/useApi.jsx:
1. Export useApi() hook returning { get, post, put, del, loading, error }
2. Base URL: "/api/admin" (same-origin)
3. All methods: fetch with credentials: "include" (sends admin.sid cookie)
4. Headers: Content-Type "application/json" for POST/PUT/DELETE
5. Auto-parse JSON response
6. On 401 response: set window.location.hash = "#/login" (session expired)
7. Return response data or throw error with response body
8. Track loading state (boolean) and last error

Create public/admin/hooks/useSocket.jsx:
1. Export useSocket() hook returning { stats, slowStats, connected }
2. On mount (if admin is authenticated): connect to Socket.IO namespace "/admin" with withCredentials: true
3. Listen for "stats" event → update stats state
4. Listen for "slowStats" event → update slowStats state
5. Listen for "disconnect" → set connected=false
6. Listen for "connect" → set connected=true
7. Cleanup: disconnect on unmount
8. Do not connect if user is not authenticated (check for adminUser prop or context)

Create public/admin/i18n/admin.js:
1. Export object with EN and VI translations for all admin UI strings
2. Structure: { en: { nav: {}, dashboard: {}, users: {}, matches: {}, content: {}, moderation: {}, ops: {}, common: {} }, vi: { ... } }
3. Include translations for: navigation items, button labels (save, cancel, delete, ban, etc.), table headers, toast messages, confirmation prompts, form labels, empty states, error messages
4. Export useI18n() hook: reads language from localStorage "admin-lang" (default "en"), returns { t(key), lang, setLang }
5. t(key) resolves dot-notation key (e.g., "nav.dashboard") against current language object
</action>

<acceptance_criteria>
- useApi().get("/users") sends GET /api/admin/users with credentials
- 401 responses automatically redirect to login page
- useSocket() connects to /admin namespace and receives real-time stats
- Socket disconnects cleanly on component unmount
- i18n covers all admin UI strings in both English and Vietnamese
- useI18n().t("nav.dashboard") returns "Dashboard" in English or "Bảng điều khiển" in Vietnamese
- Language toggle persists in localStorage
</acceptance_criteria>
