# Plan 02: Admin auth & middleware

## Objective
Implement admin authentication (login/logout), session management with separate cookie, RBAC middleware, IP allowlist, rate limiting, CSRF protection, and audit logging helpers.

## must_haves
- [ ] Admin login endpoint authenticates via email/password and verifies admin_roles membership
- [ ] Separate admin session cookie ("admin.sid") with 2-hour TTL scoped to /admin path
- [ ] requireRole(minRole) middleware factory enforces role hierarchy (super_admin > admin > moderator)
- [ ] IP allowlist middleware reads ADMIN_ALLOWED_IPS env var
- [ ] Audit log helper inserts into admin_audit_log on every admin action
- [ ] Rate limiting on login (5/15min per IP) and API (60/min)

## Tasks

### Task 1: Create admin-auth.js with session, RBAC, and audit helpers

<read_first>
- admin-auth.js
- server.js
- db.js
</read_first>

<action>
Create admin-auth.js exporting the following:

1. adminSessionMiddleware — an express-session instance configured with:
   - store: new pgSession({ pool (imported from db.js), tableName: "admin_sessions" })
   - name: "admin.sid"
   - secret: process.env.ADMIN_SESSION_SECRET || process.env.SESSION_SECRET
   - resave: false, saveUninitialized: false, rolling: true
   - cookie: { httpOnly: true, secure: "auto", sameSite: "strict", maxAge: 2 * 60 * 60 * 1000, path: "/admin" }

2. loadAdminUser(req, res, next) middleware — reads req.session.adminUserId, queries admin_roles for active role (WHERE user_id=$1 AND revoked_at IS NULL), sets req.adminUser = { id, role } or leaves undefined. Always calls next().

3. requireAdmin(req, res, next) — if !req.adminUser, respond 401 JSON { error: "ADMIN_AUTH_REQUIRED" }. Otherwise next().

4. requireRole(minRole) — middleware factory. Define ROLE_HIERARCHY = { super_admin: 3, admin: 2, moderator: 1 }. If !req.adminUser return 401. Compare userLevel = ROLE_HIERARCHY[req.adminUser.role] || 0 against requiredLevel = ROLE_HIERARCHY[minRole] || 999. If userLevel < requiredLevel return 403 JSON { error: "INSUFFICIENT_ROLE" }. Otherwise next().

5. ipAllowlist(req, res, next) — reads process.env.ADMIN_ALLOWED_IPS. If not set, calls next(). Otherwise splits by comma, trims each, checks req.ip against the list. If not included, returns 403 JSON { error: "IP_NOT_ALLOWED" }.

6. csrfJsonCheck(req, res, next) — for POST/PUT/PATCH/DELETE methods, check req.is("json"). If not JSON content-type, return 415 JSON { error: "JSON_REQUIRED" }. Otherwise next().

7. logAdminAction(adminId, action, req, targetType, targetId, details) — async function that INSERT INTO admin_audit_log (admin_id, action, target_type, target_id, details, ip, user_agent) VALUES ($1,$2,$3,$4,$5,$6,$7) using pool.query. IP from req.ip, user_agent from req.get("user-agent"). Wraps in try/catch; on error console.error("[audit] log failed:", e.message) — never throws.

8. adminLoginLimiter — RateLimiterMemory instance with points:5, duration:900 (15 min), blockDuration:900.

9. adminApiLimiter — RateLimiterMemory instance with points:60, duration:60.

10. adminRateLimitMiddleware(req, res, next) — consumes 1 point from adminApiLimiter keyed by req.ip. On rejection return 429 JSON { error: "RATE_LIMITED" }. On success next().

Export all as module.exports = { adminSessionMiddleware, loadAdminUser, requireAdmin, requireRole, ipAllowlist, csrfJsonCheck, logAdminAction, adminLoginLimiter, adminApiLimiter, adminRateLimitMiddleware }.
</action>

<acceptance_criteria>
- admin-auth.js exports all 10 listed items
- adminSessionMiddleware uses cookie name "admin.sid" with path "/admin" and maxAge 7200000
- requireRole("admin") returns 403 for a moderator, allows admin and super_admin
- requireRole("super_admin") returns 403 for admin and moderator
- ipAllowlist passes through when ADMIN_ALLOWED_IPS is unset
- logAdminAction never throws (errors are caught and logged)
- adminLoginLimiter has points=5, duration=900
- csrfJsonCheck returns 415 for non-JSON POST requests
</acceptance_criteria>

### Task 2: Add admin login/logout endpoints and mount admin routes in server.js

<read_first>
- server.js
- admin-auth.js

</read_first>

<action>
In server.js, add the admin route mounting after the existing static middleware and before the Socket.IO handlers:

1. Import from admin-auth.js: adminSessionMiddleware, loadAdminUser, requireAdmin, ipAllowlist, csrfJsonCheck, logAdminAction, adminLoginLimiter, adminRateLimitMiddleware.

2. Add admin static serving BEFORE the existing express.static("dist") line:
   app.use("/admin", express.static(path.join(__dirname, "dist/admin")));

3. Create an Express Router for admin API:
   const adminRouter = express.Router();

4. Mount the admin API with middleware chain:
   app.use("/api/admin", express.json(), ipAllowlist, adminSessionMiddleware, csrfJsonCheck, adminRouter);

5. On adminRouter, add POST "/login" handler:
   - Consume adminLoginLimiter by req.ip; on reject return 429 { error: "RATE_LIMITED" }
   - Extract email, password from req.body
   - Call verifyEmailLogin(email, password) from db.js
   - If result.error, call logAdminAction with action "auth.login_failed" (use null for adminId since login failed, store email in details), return 401 { error: "AUTH_FAILED" }
   - Query admin_roles: SELECT role FROM admin_roles WHERE user_id=$1 AND revoked_at IS NULL
   - If no role found, return 403 { error: "NOT_ADMIN" }
   - Set req.session.adminUserId = user.id
   - Call logAdminAction(user.id, "auth.login", req, "user", String(user.id), null)
   - Return 200 { ok: true, user: { id: user.id, display_name: user.display_name, role: row.role } }

6. On adminRouter, use loadAdminUser middleware for all routes after /login:
   adminRouter.use(loadAdminUser);

7. Add POST "/logout" handler (after loadAdminUser):
   - If req.adminUser, call logAdminAction(req.adminUser.id, "auth.logout", req, null, null, null)
   - Destroy session: req.session.destroy()
   - Return 200 { ok: true }

8. Add GET "/me" handler (after loadAdminUser, with requireAdmin):
   adminRouter.get("/me", requireAdmin, (req, res) => { ... })
   - Query users table for display_name by req.adminUser.id
   - Return { id, display_name, role: req.adminUser.role }

9. After requireAdmin-protected "/me", apply adminRateLimitMiddleware to all subsequent admin routes:
   adminRouter.use(requireAdmin, adminRateLimitMiddleware);

10. Add SPA fallback route for admin HTML:
    app.get("/admin/*", (req, res) => res.sendFile(path.join(__dirname, "dist/admin/index.html")));
    (Place after api/admin mount but ensure it doesn't conflict with static)
</action>

<acceptance_criteria>
- POST /api/admin/login with valid admin email+password returns 200 with user object including role
- POST /api/admin/login with valid user who is NOT admin returns 403 { error: "NOT_ADMIN" }
- POST /api/admin/login with wrong password returns 401 { error: "AUTH_FAILED" }
- POST /api/admin/login after 5 failed attempts from same IP returns 429 { error: "RATE_LIMITED" }
- POST /api/admin/logout destroys session and returns { ok: true }
- GET /api/admin/me with valid session returns user info + role
- GET /api/admin/me without session returns 401
- All login attempts (success and failure) are logged to admin_audit_log
- /admin static files served from dist/admin/
- Admin SPA fallback serves dist/admin/index.html for /admin/* paths
</acceptance_criteria>
