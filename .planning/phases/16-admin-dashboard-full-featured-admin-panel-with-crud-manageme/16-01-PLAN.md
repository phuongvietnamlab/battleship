# Plan 01: Database migrations + CLI bootstrap

## Objective
Create the 009_admin.sql migration with all admin-related tables and a CLI script to bootstrap the first super_admin user.

## must_haves
- [ ] Migration 009_admin.sql creates all 9 admin tables with correct schema and indexes
- [ ] CLI script promotes an existing user (by email) to super_admin
- [ ] npm script "admin:create" is registered in package.json

## Tasks

### Task 1: Create migration file 009_admin.sql

<read_first>
- migrations/008_premium_emojis.sql
- db.js
</read_first>

<action>
Create migrations/009_admin.sql containing CREATE TABLE IF NOT EXISTS statements for:

1. admin_roles — columns: id SERIAL PRIMARY KEY, user_id INTEGER NOT NULL REFERENCES users(id) UNIQUE, role TEXT NOT NULL CHECK (role IN ('super_admin', 'admin', 'moderator')), granted_by INTEGER REFERENCES users(id), granted_at TIMESTAMPTZ NOT NULL DEFAULT now(), revoked_at TIMESTAMPTZ. Index: idx_admin_roles_user_id on (user_id) WHERE revoked_at IS NULL.

2. admin_sessions — columns: "sid" VARCHAR NOT NULL COLLATE "default" PRIMARY KEY, "sess" JSON NOT NULL, "expire" TIMESTAMP(6) NOT NULL, "user_id" INTEGER. Indexes: idx_admin_sessions_expire on ("expire"), idx_admin_sessions_user_id on ("user_id"). Must match connect-pg-simple expected schema.

3. admin_audit_log — columns: id SERIAL PRIMARY KEY, admin_id INTEGER NOT NULL REFERENCES users(id), action TEXT NOT NULL, target_type TEXT, target_id TEXT, details JSONB, ip INET NOT NULL, user_agent TEXT, created_at TIMESTAMPTZ NOT NULL DEFAULT now(). Indexes: idx_audit_admin_id on (admin_id, created_at DESC), idx_audit_target on (target_type, target_id, created_at DESC), idx_audit_action on (action, created_at DESC), idx_audit_created_at on (created_at DESC).

4. announcements — columns: id SERIAL PRIMARY KEY, title_en TEXT NOT NULL, title_vi TEXT NOT NULL, body_en TEXT, body_vi TEXT, type TEXT NOT NULL DEFAULT 'info' CHECK (type IN ('info', 'warning', 'maintenance', 'event')), start_at TIMESTAMPTZ NOT NULL DEFAULT now(), end_at TIMESTAMPTZ, active BOOLEAN NOT NULL DEFAULT true, created_by INTEGER NOT NULL REFERENCES users(id), created_at TIMESTAMPTZ NOT NULL DEFAULT now(), updated_at TIMESTAMPTZ NOT NULL DEFAULT now(). Index: idx_announcements_active on (active, start_at, end_at).

5. reports — columns: id SERIAL PRIMARY KEY, reporter_id INTEGER NOT NULL REFERENCES users(id), reported_id INTEGER NOT NULL REFERENCES users(id), match_id INTEGER REFERENCES matches(id), reason TEXT NOT NULL CHECK (reason IN ('chat_abuse', 'cheating', 'harassment', 'other')), description TEXT, status TEXT NOT NULL DEFAULT 'pending' CHECK (status IN ('pending', 'reviewed', 'resolved', 'dismissed')), reviewed_by INTEGER REFERENCES users(id), reviewed_at TIMESTAMPTZ, resolution TEXT, created_at TIMESTAMPTZ NOT NULL DEFAULT now(). Indexes: idx_reports_status on (status, created_at DESC), idx_reports_reported_id on (reported_id).

6. chat_logs — columns: id SERIAL PRIMARY KEY, room_code TEXT NOT NULL, sender_id INTEGER REFERENCES users(id), client_id TEXT NOT NULL, message TEXT NOT NULL, flagged BOOLEAN NOT NULL DEFAULT false, created_at TIMESTAMPTZ NOT NULL DEFAULT now(). Indexes: idx_chat_logs_room on (room_code, created_at), idx_chat_logs_sender on (sender_id, created_at DESC), idx_chat_logs_flagged on (flagged) WHERE flagged = true.

7. runtime_config — columns: key TEXT PRIMARY KEY, value JSONB NOT NULL, description TEXT, updated_by INTEGER REFERENCES users(id), updated_at TIMESTAMPTZ NOT NULL DEFAULT now().

8. user_bans — columns: id SERIAL PRIMARY KEY, user_id INTEGER NOT NULL REFERENCES users(id), type TEXT NOT NULL CHECK (type IN ('ban', 'mute')), reason TEXT NOT NULL, duration INTERVAL, starts_at TIMESTAMPTZ NOT NULL DEFAULT now(), ends_at TIMESTAMPTZ, active BOOLEAN NOT NULL DEFAULT true, banned_by INTEGER NOT NULL REFERENCES users(id), unbanned_by INTEGER REFERENCES users(id), unbanned_at TIMESTAMPTZ, created_at TIMESTAMPTZ NOT NULL DEFAULT now(). Indexes: idx_user_bans_user_active on (user_id, active) WHERE active = true, idx_user_bans_ends_at on (ends_at) WHERE active = true AND ends_at IS NOT NULL.

9. daily_stats — columns: date DATE PRIMARY KEY, new_users INTEGER NOT NULL DEFAULT 0, active_users INTEGER NOT NULL DEFAULT 0, matches_played INTEGER NOT NULL DEFAULT 0, matches_classic INTEGER NOT NULL DEFAULT 0, matches_wagered INTEGER NOT NULL DEFAULT 0, points_earned BIGINT NOT NULL DEFAULT 0, points_spent BIGINT NOT NULL DEFAULT 0, points_wagered BIGINT NOT NULL DEFAULT 0, computed_at TIMESTAMPTZ NOT NULL DEFAULT now().

All CREATE TABLE and CREATE INDEX statements use IF NOT EXISTS for idempotency.
</action>

### Task 2: Create CLI bootstrap script scripts/admin-create.js

<read_first>
- scripts/admin-create.js
- db.js
</read_first>

<action>
Create scripts/admin-create.js that:

1. Reads command-line argument: the email of the user to promote (process.argv[2])
2. Validates that an email argument was provided; if not, prints usage message "Usage: npm run admin:create <email>" and exits with code 1
3. Requires db.js to get the pool and calls runMigrations(pool) first to ensure tables exist
4. Queries credentials table to find user_id WHERE type='email' AND external_id = normalizedEmail (trimmed, lowercased)
5. If no user found, prints "Error: No user found with email <email>. Register an account first." and exits code 1
6. Checks admin_roles for existing active role (WHERE user_id = found_id AND revoked_at IS NULL)
7. If role exists, prints "User <email> already has role: <role>" and exits code 0
8. INSERTs into admin_roles (user_id, role, granted_by, granted_at) VALUES (userId, 'super_admin', userId, now())
9. Prints "Success: <email> promoted to super_admin" and exits code 0
10. Uses pool.end() in finally block to close connections
11. All SQL uses parameterized queries ($1 binding)
12. Wraps main logic in async IIFE with .catch() that logs error and exits code 1
</action>

<acceptance_criteria>
- Running "node scripts/admin-create.js" with no args prints usage and exits 1
- Running with a non-existent email prints error message and exits 1
- Running with a valid registered email INSERTs into admin_roles with role='super_admin'
- Script uses parameterized queries (no string concatenation of email into SQL)
- Script calls pool.end() before exiting
- Script calls runMigrations(pool) before querying admin tables
</acceptance_criteria>
