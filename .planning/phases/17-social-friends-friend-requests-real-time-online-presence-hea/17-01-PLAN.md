# Plan 17-01: Database Migration + Friend CRUD API

## Objective

Create the `friendships` table and REST API endpoints for sending, accepting, rejecting friend requests, unfriending, and searching users by display name.

## Tasks

### Task 1: Database migration (010_friendships.sql)

Create `migrations/010_friendships.sql`:

```sql
CREATE TABLE IF NOT EXISTS friendships (
  id SERIAL PRIMARY KEY,
  user_id INTEGER NOT NULL REFERENCES users(id),
  friend_id INTEGER NOT NULL REFERENCES users(id),
  status VARCHAR(20) NOT NULL DEFAULT 'pending',
  created_at TIMESTAMPTZ DEFAULT NOW(),
  accepted_at TIMESTAMPTZ,
  UNIQUE(user_id, friend_id),
  CHECK(user_id != friend_id)
);

CREATE INDEX IF NOT EXISTS idx_friendships_user ON friendships(user_id, status);
CREATE INDEX IF NOT EXISTS idx_friendships_friend ON friendships(friend_id, status);
```

**Verify:** Server boots without error; `\dt friendships` shows the table.

### Task 2: Friend CRUD functions in db.js

Add to `db.js`:

- `sendFriendRequest(userId, targetUserId)` — INSERT with status='pending'; validate: not self, not duplicate (ON CONFLICT DO NOTHING), not blocked, count < 100
- `acceptFriendRequest(friendshipId, userId)` — UPDATE status='accepted', accepted_at=now() WHERE id=$1 AND friend_id=$2 AND status='pending'
- `rejectFriendRequest(friendshipId, userId)` — DELETE WHERE id=$1 AND friend_id=$2 AND status='pending'
- `removeFriend(userId, friendId)` — DELETE WHERE (user_id=$1 AND friend_id=$2) OR (user_id=$2 AND friend_id=$1) AND status='accepted'
- `getFriendsList(userId)` — SELECT friends with status='accepted' + JOIN users for display_name, avatar_url
- `getPendingRequests(userId)` — SELECT WHERE friend_id=$1 AND status='pending' + JOIN users
- `searchUsers(query, excludeUserId)` — SELECT from users WHERE display_name ILIKE '%query%' LIMIT 10 (exclude self)
- `getFriendshipStatus(userId, targetId)` — check if friendship exists and its status

**Verify:** Unit tests with Vitest (test/friends.test.js) — at minimum test sendFriendRequest validation (no self-add, max 100 guard).

### Task 3: REST API routes

Add Express routes (in server.js or new `routes/friends.js` if modular):

```
GET    /api/friends          → getFriendsList(userId) — requires auth
GET    /api/friends/pending  → getPendingRequests(userId) — requires auth
GET    /api/friends/search?q=name → searchUsers(q, userId) — requires auth
POST   /api/friends/request  → sendFriendRequest(userId, body.targetUserId)
POST   /api/friends/accept   → acceptFriendRequest(body.friendshipId, userId)
POST   /api/friends/reject   → rejectFriendRequest(body.friendshipId, userId)
DELETE /api/friends/:friendId → removeFriend(userId, params.friendId)
```

All routes require `socket.request.session?.user_id` (or Express session user_id). Return 401 for guests.

Rate limit: 5 requests/10s per userId on POST endpoints (reuse RateLimiterMemory pattern).

**Verify:** Manual curl/httpie test or Vitest integration test hitting the endpoints.
