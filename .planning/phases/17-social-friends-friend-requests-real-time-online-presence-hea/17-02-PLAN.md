# Plan 17-02: Real-Time Presence System (Socket.IO)

## Objective

Track authenticated users' online status (online / in-game / offline) via Socket.IO and broadcast presence changes to their accepted friends in real-time.

## Tasks

### Task 1: Server-side presence tracking data structures

Add to server.js (near the top, after rooms/queues maps):

```javascript
// Phase 17: Presence tracking
const userSockets = new Map();      // userId → Set<socketId>
const userPresence = new Map();     // userId → 'online' | 'in-game' | 'offline'
const disconnectTimers = new Map(); // userId → setTimeout handle
```

### Task 2: Presence on socket connect

In `io.on("connection")` handler, after `socket.data.userId` is set:

```javascript
if (userId) {
  // Cancel any pending offline timer
  if (disconnectTimers.has(userId)) {
    clearTimeout(disconnectTimers.get(userId));
    disconnectTimers.delete(userId);
  }
  // Register socket
  if (!userSockets.has(userId)) userSockets.set(userId, new Set());
  userSockets.get(userId).add(socket.id);
  // Set presence online (only if not already in-game from another tab)
  if (userPresence.get(userId) !== 'in-game') {
    userPresence.set(userId, 'online');
  }
  // Broadcast to online friends
  broadcastPresenceToFriends(userId, userPresence.get(userId));
  // Send friend list with current presence to this socket
  sendFriendListWithPresence(userId, socket);
}
```

### Task 3: Presence on socket disconnect

In the existing disconnect handler, add after queue cleanup:

```javascript
if (socket.data.userId) {
  const uid = socket.data.userId;
  const sockets = userSockets.get(uid);
  if (sockets) {
    sockets.delete(socket.id);
    if (sockets.size === 0) {
      userSockets.delete(uid);
      // 30s grace period before going offline
      const timer = setTimeout(() => {
        userPresence.set(uid, 'offline');
        disconnectTimers.delete(uid);
        broadcastPresenceToFriends(uid, 'offline');
      }, 30000);
      disconnectTimers.set(uid, timer);
    }
  }
}
```

### Task 4: Presence on game state changes

- When both players ready (game starts): set `userPresence.set(userId, 'in-game')` for both players → broadcast
- When game ends (gameOver, leave, disconnect forfeit): set `userPresence.set(userId, 'online')` → broadcast
- Hook into existing `placeShips` (both ready check) and `endGame`/`leaveRoom` paths

### Task 5: Helper functions

```javascript
async function broadcastPresenceToFriends(userId, status) {
  const friends = await getFriendsList(userId); // from db.js
  for (const friend of friends) {
    const friendSockets = userSockets.get(friend.id);
    if (friendSockets) {
      for (const sid of friendSockets) {
        io.to(sid).emit('friend:status-change', { userId, status });
      }
    }
  }
}

async function sendFriendListWithPresence(userId, socket) {
  const friends = await getFriendsList(userId);
  const friendsWithPresence = friends.map(f => ({
    ...f,
    status: userPresence.get(f.id) || 'offline'
  }));
  socket.emit('friend:list', friendsWithPresence);
  // Also send pending requests
  const pending = await getPendingRequests(userId);
  socket.emit('friend:pending', pending);
}
```

### Task 6: Socket.IO events for friend request notifications

When a friend request is sent (via REST API or socket event):
- Emit `friend:request-received` to target's sockets (if online)

When a friend request is accepted:
- Emit `friend:online` to both parties (if both online)

## Success Criteria

- [ ] Authenticated user connecting → friends see them come online
- [ ] User disconnecting → 30s grace → friends see them go offline
- [ ] Page refresh within 30s → no offline flicker
- [ ] Multi-tab: closing one tab doesn't trigger offline if others remain
- [ ] Game start → friends see "in-game" status
- [ ] Game end → friends see "online" status
- [ ] Friend list sent on connect includes current presence of each friend
