# Plan 12-01: Unified Quick Play with conditional stake popup

## Overview

Merge the "Wagered Match" and "Quick Play" flows into a single hero CTA. Guest users go straight to the free queue; logged-in users see a stake selection bottom sheet before joining.

## Scope

- `public/app.jsx` ÔÇö Lobby component (remove wager-strip, add stake BottomSheet, unify handler)
- `public/app.jsx` ÔÇö App component (merge `handleWageredMatch` into `handleQuickMatch`)

No server changes. No CSS changes needed (reuses existing `.wager-strip`, `.chip`, `BottomSheet` styles).

## Tasks

### Task 1: Remove wager-strip and onWageredMatch prop

**File:** `public/app.jsx`

1. In the `Lobby` function signature, remove `onWageredMatch` prop
2. Remove the entire `{authUser && balance !== null && (...)}` wager-strip block (the `<div className="wager-strip">` section with balance, chips, and wagered match button)
3. Remove the `selectedStake` state from Lobby (it will move to the new BottomSheet)

### Task 2: Add Stake Selection BottomSheet to Lobby

**File:** `public/app.jsx`

1. Add state: `const [stakeSheetOpen, setStakeSheetOpen] = useState(false);`
2. Add local state inside the sheet or Lobby: `const [sheetStake, setSheetStake] = useState(0);`
3. Modify the Quick Play hero CTA `onClick`:
   - If `authUser` is truthy: `setStakeSheetOpen(true)` (open popup)
   - If `authUser` is falsy (guest): `onQuickMatch(0)` directly (free play, no popup)
4. Add a new `<BottomSheet>` after the existing friend/bot sheets:
   ```jsx
   <BottomSheet open={stakeSheetOpen} onClose={() => setStakeSheetOpen(false)} title={t("queue.stakeSelect")}>
     <div className="wager-balance-row">­čĺ░ {balance}</div>
     <div className="wager-chips">
       {[0, 10, 25, 50, 100].map((s) => (
         <button key={s} className={"chip" + (sheetStake === s ? " active" : "")}
           onClick={() => setSheetStake(s)} disabled={s > 0 && balance < s}>
           {s === 0 ? t("queue.stake0") : s}
         </button>
       ))}
     </div>
     <button className="btn primary" onClick={() => { onQuickMatch(sheetStake); setStakeSheetOpen(false); }}
       disabled={sheetStake > 0 && balance < sheetStake}>
       {t("queue.quickMatch")}
     </button>
   </BottomSheet>
   ```

### Task 3: Unify handleQuickMatch to accept stake parameter

**File:** `public/app.jsx`

1. Change `handleQuickMatch()` to `handleQuickMatch(stake = 0)`:
   - If `stake > 0`: emit `joinQueue` with `{ type: "wagered", stake, clientId, profile }` (same as old `handleWageredMatch`)
   - If `stake === 0`: emit `joinQueue` with `{ type: "free", clientId, profile }` (existing behavior)
2. Set `queueType`, `queueStake`, `setStake` accordingly based on stake value
3. Remove the standalone `handleWageredMatch` function entirely
4. In the App render, remove `onWageredMatch={handleWageredMatch}` from `<Lobby>` props
5. Pass `onQuickMatch={handleQuickMatch}` (already done, just ensure signature matches)

### Task 4: Verify and clean up

1. Ensure the Quick Play hero button still works for guests (direct free queue join)
2. Ensure logged-in users see the stake popup and can select + confirm
3. Ensure the friends room stake selector (in FriendSheet) is untouched
4. Remove any dead code references to the old wagered match flow
