# Phase 7, Plan 03: Wagering System + Queue Replacement

**Depends on:** Plan 01 (wallet helpers), Plan 02 (queues renamed to free/wagered, ranked code removed)
**Estimated tasks:** 6

## Objective

Implement the full wagering flow: wager debit on queue join or room creation, refund on queue leave / room cancellation / mutual abandon, and winner payout on match end. Replace the old queue pairing logic with stake-level matching for the wagered queue.

## Tasks

### Task 1: Implement wagered queue join with balance debit

**Type:** code
**File(s):** `server.js` (modify)

**Actions:**
1. Import `debitWallet`, `creditWallet`, `getWalletBalance` from `./db` in the top-level require (add to the existing destructure).

2. Add a constant for valid stake presets near the top of server.js (after the queue maps):
   ```javascript
   const VALID_STAKES = [10, 25, 50, 100];
   ```

3. Rewrite the `joinQueue` handler to support wagered queue with balance debit. The handler should:
   - For `type === "free"`: allow anyone (guest or signed-in), no balance check, entry has `stake: 0`
   - For `type === "wagered"`:
     - Guard: must be signed in (`socket.data.userId != null`) → error `WAGERED_REQUIRES_ACCOUNT`
     - Guard: `arg.stake` must be in `VALID_STAKES` → error `INVALID_STAKE`
     - Read balance and debit immediately via `debitWallet(userId, stake, 'wager_debit', referenceId)` where referenceId = `"queue_" + Date.now() + "_" + socket.id`
     - If debit fails → error `INSUFFICIENT_BALANCE`
     - Store `stake` and `wagerId` (the referenceId) on the queue entry

   Updated entry shape for wagered:
   ```javascript
   const entry = {
     socketId: socket.id,
     clientId,
     queueKey,
     userId: socket.data.userId,
     stake: stake,
     wagerId: referenceId, // for refund tracking
     enqueuedAt: Date.now(),
     pairing: false,
     profile: sanitizeProfile(arg && arg.profile),
     queueType: "wagered",
   };
   ```

4. Store `socket.data.queueStake` and `socket.data.queueWagerId` on the socket for refund on disconnect/leave.

**Acceptance criteria:**
- [ ] Free queue allows any player, no balance check
- [ ] Wagered queue requires signed-in user + valid stake + sufficient balance
- [ ] Wager is debited at the moment of queue join (not later)
- [ ] Entry carries `stake` field used by `findPair` for same-stake matching
- [ ] Socket.data carries stake + wagerId for refund handling

### Task 2: Implement wager refund on queue leave and disconnect

**Type:** code
**File(s):** `server.js` (modify)

**Actions:**
1. Update the `leaveQueue` handler to refund wager when leaving a wagered queue:
   ```javascript
   socket.on("leaveQueue", async (arg, cb) => {
     if (socket.data.code) return cb && cb({ ok: true });
     const queueKey = socket.data.queueKey || queueKeyFor(socket);
     for (const type of ["free", "wagered"]) {
       const e = queues[type].get(queueKey);
       if (e && e.socketId === socket.id) {
         queues[type].delete(queueKey);
         // Refund wager if leaving wagered queue
         if (e.stake && e.userId) {
           creditWallet(e.userId, e.stake, "wager_refund", e.wagerId).catch(() => {});
           socket.emit("balanceUpdate", { balance: await getWalletBalance(e.userId).catch(() => 0) });
         }
       }
     }
     socket.data.queueType = null;
     socket.data.queueKey = null;
     socket.data.queueClientId = null;
     socket.data.queueStake = null;
     socket.data.queueWagerId = null;
     cb && cb({ ok: true });
   });
   ```

2. Update the `disconnect` handler — in the `removeFromQueues` call at the top of the disconnect handler, replace it with a refund-aware removal:
   ```javascript
   // QUEUE-03: queue cleanup with wager refund
   const queueKey = socket.data.queueKey || queueKeyFor(socket);
   for (const type of ["free", "wagered"]) {
     const e = queues[type].get(queueKey);
     if (e && e.socketId === socket.id) {
       queues[type].delete(queueKey);
       if (e.stake && e.userId) {
         creditWallet(e.userId, e.stake, "wager_refund", e.wagerId).catch(() => {});
       }
     }
   }
   ```

3. Update the D-11 re-queue path (disconnect before game starts for queue-matched rooms): the survivor was already debited; carry `stake` and `wagerId` onto the re-queued entry so they don't lose their wager tracking.

**Acceptance criteria:**
- [ ] Leaving wagered queue refunds the player's stake immediately
- [ ] Disconnecting while in wagered queue triggers a refund
- [ ] Free queue leave/disconnect has no wallet interaction
- [ ] D-11 re-queue preserves stake + wagerId on survivor entry

### Task 4: Implement wager payout on match end

**Type:** code
**File(s):** `server.js` (modify), `db.js` (modify)

**Actions:**
1. Modify `recordMatch` in db.js to accept a `stake` parameter:
   ```javascript
   async function recordMatch(winnerId, loserId, reason, mode, startedAt, stake = 0)
   ```
   - Add `stake` to the INSERT: `INSERT INTO matches (winner_id, loser_id, reason, mode, started_at, ended_at, stake) VALUES ($1,$2,$3,$4,$5,now(),$6)`
   - After the match INSERT (still inside the same transaction), if `stake > 0` and `winnerId` is not null:
     ```javascript
     if (stake > 0 && winnerId != null) {
       const pot = stake * 2;
       const payout = Math.floor(pot * 0.9); // 90% to winner, 10% system fee
       // Credit winner
       const { rows: wRows } = await client.query(
         "SELECT balance FROM wallets WHERE user_id = $1 FOR UPDATE", [winnerId]
       );
       if (wRows.length > 0) {
         const newBal = wRows[0].balance + payout;
         await client.query("UPDATE wallets SET balance = $1, updated_at = now() WHERE user_id = $2", [newBal, winnerId]);
         await client.query(
           "INSERT INTO transactions (user_id, type, amount, balance_after, reference_id) VALUES ($1, 'wager_win', $2, $3, $4)",
           [winnerId, payout, newBal, matchCode]
         );
       }
     }
     ```
   - Generate a `matchCode` for the reference_id (e.g., `"match_" + Date.now()`) or use the room code passed as an extra parameter.

2. Update all `recordMatch` call sites in server.js to pass `room.stake || 0` as the 6th argument:
   - `scheduleSeatRelease` disconnect path
   - `endGameForfeit`
   - `doShot` win path
   - `leaveRoom` handler

3. After `recordMatch` completes (or in the gameOver emit path), push `balanceUpdate` to the winner:
   ```javascript
   // After gameOver emits, push balance to winner
   if (room.stake > 0 && winnerId) {
     const wUserId = room.players[winnerId]?.userId;
     if (wUserId) {
       getWalletBalance(wUserId).then((bal) => {
         emitToClient(room, winnerId, "balanceUpdate", { balance: bal });
       }).catch(() => {});
     }
   }
   ```

**Acceptance criteria:**
- [ ] `recordMatch` records the stake column in the matches table
- [ ] Winner receives 90% of pot (floor of `stake * 2 * 0.9`) in a single transaction with match INSERT
- [ ] System fee (10%) is implicit — never credited to anyone
- [ ] Winner gets a `balanceUpdate` event after game ends
- [ ] 0-stake games skip all wallet operations

### Task 5: Handle wagered match edge cases

**Type:** code
**File(s):** `server.js` (modify)

**Actions:**
1. **Mutual abandon (sweepRooms)**: When `sweepRooms` evicts a room with `room.stake > 0` where the game was started but not recorded, refund both players:
   ```javascript
   function sweepRooms() {
     const now = Date.now();
     for (const code in rooms) {
       const r = rooms[code];
       if (r.order.length === 0) { delete rooms[code]; continue; }
       if (r.lastActivityAt && now - r.lastActivityAt > ROOM_IDLE_THRESHOLD_MS) {
         clearTurnTimer(r);
         // Refund wagers if game started but never resolved
         if (r.stake > 0 && r.started && !r.recorded) {
           for (const id of r.order) {
             const userId = r.players[id]?.userId;
             if (userId) creditWallet(userId, r.stake, "wager_refund", "sweep_" + code).catch(() => {});
           }
         }
         delete rooms[code];
       }
     }
   }
   ```

2. **Wagered queue createMatchedRoom**: When a wagered match is created, carry the stake and wagerId from both entries onto the room:
   ```javascript
   // In createMatchedRoom, for wagered type:
   if (type === "wagered") {
     rooms[code].stake = entryA.stake; // both have same stake (findPair guarantees)
   }
   ```
   Also carry `wagerId` onto each player seat for potential refund tracking.

3. **Refund on pre-start disconnect (D-11 path)**: When a wagered queue-matched room is torn down before start (opponent disconnected), refund the disconnecting player's wager:
   ```javascript
   // The disconnecting player's wager needs refund
   const dcUserId = room.players[clientId]?.userId;
   if (room.stake > 0 && dcUserId) {
     creditWallet(dcUserId, room.stake, "wager_refund", "prestart_dc_" + code).catch(() => {});
   }
   ```
   The survivor is re-queued with their original entry (wager already debited, still tracked).

**Acceptance criteria:**
- [ ] Mutual abandon (idle sweep) refunds both players
- [ ] Queue-matched wagered games carry stake onto the room
- [ ] Pre-start disconnect refunds the disconnecting player
- [ ] Survivor's wager remains debited (they're re-queued to play)
