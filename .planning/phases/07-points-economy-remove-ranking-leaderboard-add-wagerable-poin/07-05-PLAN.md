# Phase 7, Plan 05: Client UI + Integration

**Depends on:** Plan 02 (ranked UI removed), Plan 03 (wagering system), Plan 04 (purchase system)
**Estimated tasks:** 5

## Objective

Build the client-side UI for the points economy: stake selector in the lobby, balance display, wagered queue flow, and the in-match power-up purchase button. Wire all new socket events to the React UI.

   ```javascript
   "wallet.insufficientBalance": "Không đủ điểm",
   "wallet.zeroBalance": "Chơi miễn phí hoặc thắng cược để kiếm điểm!",
   ```

3. In the `App` component, add state for balance:
   ```javascript
   const [balance, setBalance] = useState(null); // null = guest/unknown, number = signed-in
   ```

4. Listen for `balanceUpdate` on the socket:
   ```javascript
   useEffect(() => {
     if (!socket) return;
     const handleBalance = (data) => setBalance(data.balance);
     socket.on("balanceUpdate", handleBalance);
     return () => socket.off("balanceUpdate", handleBalance);
   }, [socket]);
   ```

5. Fetch balance on auth state change (after successful login or on mount if `authUser`):
   ```javascript
   useEffect(() => {
     if (authUser) {
       fetch("/api/wallet").then(r => r.json()).then(d => setBalance(d.balance)).catch(() => {});
     } else {
       setBalance(null);
     }
   }, [authUser]);
   ```

6. Display balance in the lobby header area (visible when `authUser` is not null):
   ```jsx
   {balance !== null && (
     <div className="wallet-balance" aria-label={t("wallet.balance")}>
       🪙 {balance} {t("wallet.balance")}
     </div>
   )}
   ```

**Acceptance criteria:**
- [ ] Balance state is initialized from `GET /api/wallet` on auth
- [ ] Balance updates reactively via `balanceUpdate` socket event
- [ ] Balance is displayed in the lobby for signed-in users
- [ ] Guests see no balance indicator
- [ ] Balance display updates after wager debit, win, or purchase

   ```javascript
   function handleWageredMatch(stake) {
     socket.emit("joinQueue", { type: "wagered", stake, clientId, profile }, (res) => {
   ```

3. Add a `PowerupShop` component that shows during advance-mode battles for signed-in players:
   ```jsx
   function PowerupShop({ socket, stake, balance, purchasesRemaining, onClose }) {
     const price = Math.max(5, Math.round(0.15 * stake));
     const [buying, setBuying] = useState(false);

     function handleBuy(type) {
       if (buying || purchasesRemaining <= 0 || balance < price) return;
       setBuying(true);
       socket.emit("buyPowerup", { type }, (res) => {
         setBuying(false);
         if (!res.ok) { /* show error toast */ }
       });
     }

     return (
       <div className="powerup-shop">
         <h3>{t("shop.title")}</h3>
         <div className="shop-info">{t("shop.price", { n: price })} · {t("shop.remaining", { n: purchasesRemaining })}</div>
         <div className="shop-grid">
           {POWERS.map(type => (
             <button key={type} className="btn ghost shop-item" onClick={() => handleBuy(type)}
               disabled={buying || purchasesRemaining <= 0 || balance < price}>
               {t("pw." + type)} — {price} 🪙
             </button>
           ))}
         </div>
         <button className="btn ghost" onClick={onClose}>{t("common.close") || "×"}</button>
       </div>
     );
   }
   ```

4. Add a shop toggle button during battle phase (only visible when mode=advance and user is signed in):
   ```jsx
   {mode === "advance" && userId && (
     <button className="btn ghost shop-toggle" onClick={() => setShowShop(!showShop)}>
       🛒 {purchasesRemaining > 0 ? `(${purchasesRemaining})` : "—"}
     </button>
   )}
   {showShop && <PowerupShop ... />}
   ```

5. Listen for `oppBoughtPowerup` event and show a brief notification:
   ```javascript

     // Show a brief toast/notification: "Opponent purchased a power-up!"
     setOppBoughtNotice(true);
     setTimeout(() => setOppBoughtNotice(false), 2000);
   });
   ```

6. Track `purchasesRemaining` in component state, initialized from `syncPayload.purchasesRemaining` and decremented on each successful buy.

**Acceptance criteria:**
- [ ] Shop button visible during advance-mode battles for signed-in players
- [ ] Shop shows all 5 power-up types with the calculated price
- [ ] Buy button disabled when balance < price or cap reached
- [ ] Successful purchase updates inventory and balance reactively

### Task 5: Display stake info during game and on game-over

   ```javascript
   "game.won": "+{n} điểm thắng!",
   "game.lost": "-{n} điểm đã cược",
   ```

5. When joining a room (as the second player), if the room has a stake > 0, show a confirmation prompt before the join actually happens:
   ```jsx

     if (!confirm(t("wallet.yourStake", { n: roomStake }))) return;
   }
   ```
   Note: The actual stake is visible from the server's response — the client shows it and requires the player to accept. In practice, the server deducts on joinRoom; if the player wants to leave before placing ships, they can leaveRoom and get a refund (handled server-side in Plan 03).

- [ ] Game-over screen shows winnings (winner) or loss (loser) for wagered games
- [ ] Free games (stake=0) show no wallet-related info during game
