  <action>
    Add a `MatchHistory` function component in app.jsx (same location pattern as ProfileView):

    **Props:** `{ authUser, onBack, t }`

    **State:**
    - matches: array (accumulated from pagination)
    - page: number (current page, starts at 1)
    - hasMore: boolean
    - loading: boolean
    - total: number
    - filters: { result: "all", mode: "all", wager: "all" }

    **Data fetching:**
    - `loadPage(p, filters)` → fetch `/api/matches?page=${p}&limit=20&result=...&mode=...&wager=...`
    - On page change or filter change → call loadPage
    - When filter changes → reset matches to [], page to 1
    - Append results when page > 1

    **IntersectionObserver (load-more):**
    - Sentinel div at bottom of list
    - Observer triggers setPage(p => p + 1) when sentinel is visible and hasMore && !loading

    **Date formatting:**
    - Inline `formatMatchTime(isoStr)` helper
    - < 60 min → "X phút trước" / "Xm ago"
    - < 24 hr → "X giờ trước" / "Xh ago"
    - < 7 days → "X ngày trước" / "Xd ago"
    - ≥ 7 days → "dd/MM/yyyy HH:mm"

    **Render structure:**
    ```jsx
    <div className="history-container">
      <div className="history-header">
        <button className="btn ghost" onClick={onBack}>← Back</button>
        <h2>{t("history.title")}</h2>
        <span className="history-total">{total} matches</span>
      </div>
      <div className="history-filters">
        {/* Filter pill groups for result, mode, wager */}
      </div>
      <div className="history-list">
        {matches.map(m => <MatchCard key={m.id} match={m} />)}
        <div ref={sentinelRef} className="history-sentinel" />
        {loading && <div className="history-loading">...</div>}
      </div>
      {!loading && matches.length === 0 && <EmptyState />}
    </div>
    ```

    **MatchCard sub-component (inline):**
    - Avatar placeholder (first letter of opponent name)
    - Opponent display name
    - Result badge: ✅ Thắng / ❌ Thua (green/red)
    - Points delta: +45 pts (green) / -50 pts (red)
    - Mode chip: Classic / Advance
    - Time: formatMatchTime(m.endedAt)
    - Reason indicator for disconnect/timeout (subtle icon)

    **Empty state:**
    - "Chưa có trận đấu nào. ⚓" / "No battles yet. ⚓"
    - CTA button back to lobby

    **Filter pills:**
    - 3 groups (result, mode, wager)
    - Each is a row of pill buttons
    - Active pill has filled bg, inactive is outline
    - Clicking a pill updates filters state → triggers refetch
  </action>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-05 | Tampering/XSS | opponent displayName render | mitigate | React auto-escapes JSX text content — no dangerouslySetInnerHTML used |
| T-13-06 | Information Disclosure | opponent stats popup | accept | Only aggregate stats (winRate, totalGames) shown — no PII or match list of others |
| T-13-07 | Denial of Service | Rapid filter changes | mitigate | Debounce filter changes; cancel in-flight requests on filter change |
</threat_model>
