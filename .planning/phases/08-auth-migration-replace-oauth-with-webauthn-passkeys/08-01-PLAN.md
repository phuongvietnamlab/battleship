<objective>
Remove all OAuth/Passport code and dependencies (Google, Facebook, passport) from the project. Replace Passport's user-loading mechanism with a simple express middleware that reads req.session.user_id and attaches req.user. Refactor email/password login to not depend on Passport (manual session management). Add the webauthn_credentials migration for Phase 8 Plan 02.

This is the "tear down old, prepare foundation" plan. After it lands:
- No OAuth code remains in the codebase
- No Passport dependency exists
- Email/password auth still works with pure express-session
- The webauthn_credentials table is ready for passkey storage
- Sign-out routes work without req.logout (Passport method)

Purpose: Clean removal of unused OAuth infrastructure that requires verified business accounts, and preparation for WebAuthn passkeys as the primary auth method.
Output: Modified server.js (OAuth removed, middleware added), modified db.js (getUserById), new migration, cleaned package.json.
</objective>

       ```js
       // ─── User loading middleware (replaces Passport deserializeUser) ─────────────
       // Reads user_id from session, queries user, attaches to req.user.
       // Runs on every request after sessionMiddleware.
       const { getUserById } = require("./db");
       app.use(async (req, res, next) => {
         if (req.session && req.session.user_id) {
           try {
             req.user = await getUserById(req.session.user_id);
           } catch (e) {
             req.user = null;
           }
         } else {
           req.user = null;
         }
         next();
       });
       ```

       async function getUserById(id) {

         return rows[0] || null;
       }

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| session → req.user | Session user_id used to load user object — must validate user exists |
| env cleanup | OAuth env vars (GOOGLE_*, FACEBOOK_*) become dead config — no security impact |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Spoofing | user-loading middleware | mitigate | Only reads user_id from server-side session store (connect-pg-simple); client cannot forge session data. getUserById returns null if user doesn't exist. |
| T-08-02 | Elevation | session fixation on email login | mitigate | Keep req.session.regenerate() before stamping user_id (SEC-05 preserved from Phase 2). |
| T-08-03 | Tampering | signout without req.logout | accept | Session.destroy() is sufficient — Passport's req.logout only cleared req.user in memory; session destruction is the real revocation. |
</threat_model>
