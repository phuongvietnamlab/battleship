# Phase 2 — UI Design Contract: Accounts & Identity

> Visual and interaction contract for Phase 2 frontend work. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope (updated — 3-method auth):** Five surfaces — (1) auth screen: two OAuth buttons + collapsible email/password form + forgot-password entry, (2) signed-in header avatar chip + dropdown menu, (3) profile view (own + other player, zero-state stats), (4) email verification banner, (5) password reset screen. The in-game battle screen is untouched. Source: CONTEXT.md D-12, D-21.
>
> **Update log:** Baseline was Google-only (02-01..02-04). This revision adds Facebook OAuth button, collapsible email/password form (login + signup + forgot-password), email verification hint, and password reset screen per CONTEXT.md D-16, D-17, D-19, D-21.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — custom CSS in `public/style.css` |
| Preset | not applicable |
| Component library | none — hand-rolled React components |
| Icon library | Unicode emoji / inline SVG (existing pattern in app.jsx) |
| Font | "Be Vietnam Pro" (body, labels, form inputs), "Oswald" (headings, CTA buttons, stat figures) — loaded via `public/index.html` |

**Rule:** Extend `public/style.css` only. Do NOT introduce Tailwind, shadcn, or any new CSS framework. New selectors follow the existing BEM-lite flat naming pattern (e.g. `.auth-screen`, `.google-signin-btn`, `.fb-signin-btn`, `.email-form`, `.profile-view`, `.verify-banner`). New form inputs follow the existing `.code-input` pattern: `background: rgba(255,255,255,.06); border: 1px solid rgba(150,200,255,.25); border-radius: 12px`.

## Spacing Scale

All values are the existing project 8-point scale as read from `style.css`. Extend; do not override.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, avatar inner padding, form label gap |
| sm | 8px | Dropdown item vertical padding, avatar chip gap, input focus outline offset |
| md | 16px | Lobby button vertical padding, form section spacing, profile section spacing |
| lg | 24px | Profile card padding, lobby section gaps, auth screen between-group gaps |
| xl | 32px | Profile stats row spacing, auth screen top padding |
| 2xl | 48px | Profile view top margin |
| 3xl | 64px | Not used in this phase |

**Exceptions:**
- Avatar touch target: 44px minimum (`.profile-chip` height 34px + 5px top/bottom padding). Source: WCAG 2.5.5 AA.
- Dropdown menu minimum width: 200px.
- Email form inputs: height 48px (padding 14px 16px) — matches existing `.code-input` rhythm, multiples-of-4 compliant.
- OAuth buttons: height 52px (padding 16px) — 4px taller than standard `.btn` to provide visual separation from game-action CTAs.

## Typography

Directly derived from `style.css` existing declarations. No new font sizes introduced.

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body / dropdown labels / form labels | 13px | 400 (regular) | 1.5 | "Be Vietnam Pro", system-ui |
| Meta / secondary / helper / error text | 12px | 400 (regular) | 1.5 | "Be Vietnam Pro", system-ui |
| Button / CTA label | 16px | 600 (semibold) | 1 | "Oswald" uppercase (game CTAs); "Be Vietnam Pro" mixed-case (OAuth/email buttons — brand requirement) |
| Profile stat figure | 22px | 700 (bold) | 1 | "Oswald" (matches `.pc-score`) |

**Rule:** Exactly 4 sizes (12, 13, 16, 22px) and exactly 2 weight bands: 400 (body, labels, meta) and 600–700 (Oswald headings, CTA buttons, stat figures). No intermediate weights (500) anywhere in this phase.

**Form input typography exception:** Password and email inputs use `font-family: "Be Vietnam Pro"; font-size: 16px; font-weight: 400` — 16px prevents iOS auto-zoom on focus (iOS zooms inputs < 16px). This is a functional exception, not a new type token.

## Color

All values are tokens already defined in `style.css :root`. No new color values are introduced; Phase 2 components reference only these tokens plus the two brand-required additions below.

| Role | Token / Value | Usage |
|------|---------------|-------|
| Dominant (60%) | `#04101f → #0a325f` (background gradient) | Page background — unchanged |
| Secondary (30%) | `var(--panel)` = `rgba(7, 22, 44, 0.72)` | Auth screen container, avatar chip, dropdown panel, profile card, email form background |
| Accent (10%) | `var(--gold)` = `#f0c14b` | Sign-in button (primary game CTA only — NOT OAuth buttons), avatar chip hover border, dropdown active-item left bar, profile stat figures |
| Destructive | `var(--hit)` = `#ff4d5e` | "Sign out all devices" menu item, error state text/border, weak password inline error |

**Accent reserved for (explicit list):**
1. Active/hover border on the signed-in `.profile-chip`
2. Dropdown menu active-item left indicator bar (2px left border on hovered row)
3. Profile zero-state stat numbers (wins: 0, losses: 0, games: 0)
4. Input focus ring: `border-color: var(--gold); box-shadow: 0 0 0 3px rgba(240,193,75,.25)` (matches `.code-input:focus`)

Accent is NOT used on: OAuth button backgrounds (brand-governed), generic dropdown items, avatar circle background, profile body text, metadata, email form submit button (uses `.btn.steel` variant for distinction from game CTAs).

**Brand-required color additions (not CSS variables — inline only):**
- Google button background: `rgba(255, 255, 255, 0.92)` (Google brand: white background required); text `#1f1f1f`
- Facebook button background: `#1877F2` (Meta brand blue); text `#ffffff`

These two values appear ONLY on their respective OAuth buttons. They are documented here to explain the deviation from the token system, not as new global tokens.

**Avatar fallback background:** `var(--sky)` = `#6fc3f3` with `color: #06182c` initial letter — matches existing `.avatar-fallback` pattern.

**Focal points:**
- Signed-out auth screen: two OAuth buttons are the visual anchors (brand colors provide contrast against the dark panel). The email form is recessed/collapsed.
- Signed-in lobby: the `.profile-chip` avatar in `.topbar-right` is the identity anchor.

### Surface 1: Auth Screen (home lobby entry — replaces/extends D-12 "Sign in with Google")

**Trigger:** User clicks "Sign in" affordance on the home lobby. Renders as a new screen state (`screen === 'auth'`) — not a modal overlay, not a route. Source: CONTEXT.md D-12, code_context "Screen-state-driven SPA."

**Layout — `.auth-screen` (new):**
- Max-width: 480px (matches `.app` and `.lobby`)
- Margin: `6vh auto 0` (matches `.lobby`)
- Background/border/backdrop: same as `.lobby` — glass panel (`var(--panel)`, `border: 1px solid var(--panel-brd)`, `backdrop-filter: blur(14px) saturate(1.2)`)
- Padding: 32px 30px (matches `.lobby`)
- Animation: `rise 0.55s cubic-bezier(.34,1.4,.4,1) both` (existing keyframe)

**Auth screen header:**
- Heading: "Sign in" / "Đăng nhập" — 22px Oswald weight 700 `#eaf2ff`, `margin: 0 0 6px`
- Sub: "Continue as guest to play instantly, or sign in to save your progress." / "Chơi ngay với tư cách khách, hoặc đăng nhập để lưu thành tích." — 14px `#a9ccec`, `margin: 0 0 24px`

**OAuth button group (`.oauth-btn-group`):**
- Stack direction: column, `gap: 12px`
- Two buttons: Google (top), Facebook (bottom)

**Google sign-in button (`.google-signin-btn`):**
- Width: 100%
- Height: 52px (padding: 16px top/bottom)
- Background: `rgba(255, 255, 255, 0.92)` — Google brand requirement; not the gold gradient
- Text color: `#1f1f1f`
- Border: `1px solid rgba(255,255,255,0.20)` at rest; `1px solid rgba(255,255,255,0.50)` on hover
- Border-radius: 12px (matches `.btn`)
- Font: "Be Vietnam Pro", 16px, weight 600, mixed-case — NOT Oswald uppercase (Google brand waiver)
- Google logo: inline SVG 18×18px, positioned left with 8px gap; standard Google G mark
- Hover: `filter: brightness(1.04)` (consistent with `.btn:hover`)
- Disabled: `opacity: 0.5; cursor: not-allowed` — during OAuth redirect in-flight
- `aria-label`: `t("auth.signInGoogle")`

**Facebook sign-in button (`.fb-signin-btn`):**
- Width: 100%
- Height: 52px (padding: 16px top/bottom)
- Background: `#1877F2` (Meta brand blue, required)
- Text color: `#ffffff`
- Border: `none` at rest; on hover same border as Google button but white at 20%
- Border-radius: 12px
- Font: "Be Vietnam Pro", 16px, weight 600, mixed-case — NOT Oswald uppercase (Meta brand waiver)
- Facebook logo: inline SVG "f" logomark 18×18px, left-aligned with 8px gap — white on brand blue
- Hover: `filter: brightness(1.08)`
- Disabled: `opacity: 0.5; cursor: not-allowed`
- `aria-label`: `t("auth.signInFacebook")`

**Divider (`.divider` — existing class, reused):**
- "OR" / "HOẶC" — existing pattern between OAuth buttons and email form

**Collapsible email form section (`.email-form-section`):**
- Trigger: `<button class="btn ghost" aria-expanded={open}>` labeled `t("auth.continueEmail")` — "or continue with email" / "hoặc tiếp tục bằng email"
- Width: 100%; height: 44px; padding: 10px 16px; font-size 13px; NOT Oswald, NOT uppercase — distinguishes from game CTAs
- On expand: the button transforms into a section header (collapses to a small "sign in with email" label row); the form animates in with `rise 0.3s ease both`
- `aria-controls` pointing to the form panel; `aria-expanded` toggled on click

**Email form panel (`.email-form` — new):**
- Visible only when expanded
- Padding: 0 (inside the section, not its own panel)
- Inputs follow `.code-input` visual language but with `font-size: 16px` (iOS zoom prevention), `letter-spacing: normal`, `text-transform: none`

**Email form — login mode (default):**
- Email input: `type="email"`, `placeholder="you@example.com"`, label `t("auth.emailLabel")`
- Password input: `type="password"`, `placeholder="••••••••"`, label `t("auth.passwordLabel")`; show/hide toggle (eye icon — 18px Unicode `👁`/`🙈` acceptable; or inline SVG)
- Submit button: `btn steel` (blue variant), full width, label `t("auth.signInBtn")` — "Sign in" / "Đăng nhập". Uses `.btn.steel` (not `.btn.primary`) to distinguish from game CTAs.
- "Forgot password?" link: `<button class="link-btn">` — 12px `#a9ccec`, hover: underline + `var(--sky)` color; positioned below password field, right-aligned
- Switch to signup link: `t("auth.noAccount")` — "Don't have an account? Sign up" / "Chưa có tài khoản? Đăng ký" — 12px `#a9ccec`, centered below submit button

**Email form — signup mode:**
- Same fields as login plus: Display name input (optional, label `t("auth.displayNameLabel")`, placeholder `t("auth.displayNamePlaceholder")`) — appears above email in signup mode
- Submit button: `btn steel`, label `t("auth.createAccountBtn")` — "Create account" / "Tạo tài khoản"
- Switch to login link: `t("auth.haveAccount")` — "Already have an account? Sign in" / "Đã có tài khoản? Đăng nhập"

**Form mode toggling:** `loginMode` boolean state. Toggling does NOT reset field values — email field preserves entered value between login/signup mode switches.

**Inline field validation (shown on submit attempt only, NOT on-blur):**
- Error text: 12px `color: #ffb3bb` (matches existing `.error` text color) below each field
- Error border: `border-color: var(--hit)` on the field
- Weak password: shown only when `password.length < 8`; copy `t("auth.errWeakPassword")`
- Email in use (from server): shown below email field, copy `t("auth.errEmailInUse")`
- Login failed (wrong credentials): shown above the submit button as an `.error` block, copy `t("auth.errAuthFailed")`

**Back button:**
- `<button class="btn ghost">` — `t("auth.backToLobby")` — below the form; returns to `screen === 'lobby'`

### Surface 2: Signed-in Header — Avatar Chip + Dropdown Menu

**Location:** `.topbar-right` (right side of the top bar).

**Avatar chip** (`.profile-chip` — already defined in `style.css`):
- Existing styles apply as-is: 28px avatar, 13px name, 20px border-radius, panel background
- Name: truncated at 150px max-width with `text-overflow: ellipsis`
- On mobile ≤380px: name hidden, chip collapses to avatar only (already in `style.css`)
- Cursor: `pointer`; hover: `border-color: var(--gold)`
- Active (dropdown open): `border-color: var(--gold); box-shadow: 0 0 0 2px rgba(240,193,75,0.25)`
- Unverified email badge: if `emailVerified === false`, show a 6px amber dot (`background: #f0a14b`) at top-right of the avatar circle — `position: absolute; top: 0; right: 0; width: 8px; height: 8px; border-radius: 50%`. No tooltip; the profile screen carries the full hint.

**Dropdown menu** (`.avatar-menu` — new):
- Triggered by click/tap on `.profile-chip`
- Position: `absolute; top: calc(100% + 8px); right: 0` anchored to the chip
- Width: 200px (min); auto if content wider
- Background: `var(--panel)` with `backdrop-filter: blur(14px) saturate(1.2)`
- Border: `1px solid var(--panel-brd)`
- Border-radius: 12px
- Box-shadow: `0 12px 36px rgba(2,12,30,0.55)`
- z-index: 80 (above `.overlay` at 50, below `.notice-toast` at 200)
- Animation: `rise 0.25s cubic-bezier(.34,1.4,.4,1)`
- Close on: click outside, Escape key, item selection

**Dropdown items** (`.avatar-menu-item` — new):
- Padding: 12px 16px
- Font: "Be Vietnam Pro", 13px, weight 400, color `#dbe7f3`
- Hover background: `rgba(255,255,255,0.06)`
- Hover left indicator: `border-left: 2px solid var(--gold)`; padding-left compensates by 2px
- Icon: 16px emoji prefix with 8px gap
- Separator: `border-top: 1px solid var(--panel-brd)` before destructive item group

**Dropdown item list (ordered):**
1. "View profile" / "Xem hồ sơ" — navigates to `screen === 'profile'`
2. "Sign out" / "Đăng xuất" — POST `/auth/signout`; returns to lobby as guest
3. --- separator ---
4. "Sign out all devices" / "Đăng xuất tất cả thiết bị" — color `var(--hit)`; triggers inline confirmation

### Surface 3: Profile View (screen state)

**Trigger:** "View profile" from dropdown, or future profile links by opaque user ID.

**Implementation:** New screen state `screen === 'profile'` — not a route, not a modal. Renders in place of `.lobby`.

**Layout — `.profile-view` (new):**
- Max-width: 480px; Margin: `6vh auto 0`; Background/border/backdrop: same as `.lobby` — glass panel
- Padding: 32px 32px; Animation: `rise 0.55s cubic-bezier(.34,1.4,.4,1) both`

**Profile header section (`.profile-header`):**
- Avatar: 64px circle; `object-fit: cover`; fallback: `var(--sky)` background + 28px Oswald initial letter
- Display name: 22px Oswald, weight 700, `#eaf2ff`; max-width 260px, truncated
- Member since: 12px, `#a9ccec`, formatted as `t("profile.memberSince", { month, year })`
- Layout: row — avatar left (64px), name + since stacked right; gap 16px; align-items center

**Email verification hint (`.verify-hint` — new, shown only for own profile with `emailVerified === false`):**
- Positioned below the profile header section, above stats
- Background: `rgba(240, 161, 75, 0.12)` (amber tint)
- Border: `1px solid rgba(240, 161, 75, 0.35)`
- Border-radius: 8px; padding: 10px 12px; font-size: 13px; color: `#f0c47b`
- Copy: `t("auth.verifyEmailHint")` + `<button class="link-btn">` `t("auth.resendVerify")` inline
- Shown ONLY on own profile when account was created via email/password and is unverified

**Stats row (`.profile-stats`) — zero-state scaffold:**
- 3-column grid (wins | separator | losses); equal width
- Each stat cell: label on top (12px uppercase `#a9ccec`), figure below (22px Oswald bold `var(--gold)`)
- Zero-state figure: "0" — same style as real numbers (D-10: no UI rework when Phase 3 fills data)
- "No games yet" sub-line: 13px `#a9ccec` centered below stats row; only shown when all 3 figures are 0

**Back navigation:**
- `<button class="btn ghost">` — `t("profile.back")` at bottom; width auto; padding: 8px 20px; not full-width

**Own vs. other-player profile:**
- Own profile: shows "Sign out" ghost button below stats (shortcut — same as dropdown item); shows verify hint if email unverified
- Other player profile: no sign-out; adds "Challenge" button placeholder (disabled, `opacity: 0.4`) for Phase 5 wiring
- Profile 404: `.error` block with `t("profile.notFound")`

### Surface 5: Password Reset Screen (screen state)

**Trigger:** Clicking "Forgot password?" in the email form, OR following the emailed reset link (`?token=...` URL query param on load). Two sub-states:

**Sub-state A: Reset request form (`screen === 'reset-request'`):**
- Layout: same glass panel as `.auth-screen`, max-width 480px
- Heading: `t("auth.resetHeading")` — "Reset password" / "Đặt lại mật khẩu"
- Sub: `t("auth.resetSub")` — "Enter your account email. If it exists, we'll send a reset link." / "Nhập email tài khoản. Nếu tồn tại, chúng tôi sẽ gửi link đặt lại mật khẩu."
- Email input (same style as auth screen)
- Submit: `btn steel`, `t("auth.resetSendBtn")` — "Send reset link" / "Gửi link đặt lại"
- On success (200 from server): show success state — `.error` block replaced by success message `t("auth.resetSent")` — "Check your email for a reset link. It expires in 1 hour." / "Kiểm tra email để nhận link đặt lại. Link hết hạn sau 1 giờ." — background `rgba(111,195,243,0.12)`, border `rgba(111,195,243,0.35)`, color `var(--sky)` (same pattern as existing positive feedback elsewhere)
- Rate-limit error (429): `t("auth.errRateLimited")` in `.error` block
- Back link: `t("auth.backToSignIn")` — returns to `screen === 'auth'`

**Sub-state B: New password form (`screen === 'reset-confirm'`, token from URL):**
- Triggered on page load when `?token=` query param is present — parsed before socket init
- Layout: same glass panel
- Heading: `t("auth.resetNewHeading")` — "Choose a new password" / "Chọn mật khẩu mới"
- New password input (show/hide toggle), confirm password input
- Submit: `btn steel`, `t("auth.resetConfirmBtn")` — "Set new password" / "Đặt mật khẩu mới"
- Inline validation: password < 8 chars shows `t("auth.errWeakPassword")` below field; passwords not matching shows `t("auth.errPasswordMismatch")` — validation on submit only, not on-blur
- Token invalid/expired (400 from server): `.error` block with `t("auth.errBadToken")` + `<button class="link-btn">` `t("auth.requestNewReset")` — routes back to `screen === 'reset-request'`
- On success: show `t("auth.resetSuccess")` success message; auto-redirect to `screen === 'auth'` after 2 seconds

## Interaction Contracts

### Sign-in Entry

- Signed-out lobby: "Sign in" ghost button in `.topbar-right` (or below the game-action buttons on small screens). Clicking sets `screen === 'auth'`.
- Guest lobby play: no sign-in prompt forced. AUTH-01 guarantee.

### Auth Screen Flows

**Google / Facebook OAuth:**
1. User taps OAuth button.
2. Button enters disabled state immediately.
3. Browser navigates via full-page redirect (no popup, no iframe) to `/auth/google` or `/auth/facebook`.
4. On return: `?authError=1` → show `.error` block with `t("auth.errFailed")`; button re-enables.
5. On success: React fetches `/api/me`; transitions to signed-in state; avatar chip appears in `.topbar-right`.

**Email sign-in:**
1. User expands email form section.
2. User fills email + password; clicks "Sign in".
3. While request in-flight: submit button disabled + `opacity: 0.5`.
4. On `AUTH_FAILED` from server: `.error` block with `t("auth.errAuthFailed")` above submit button.
5. On `RATE_LIMITED` (429): `.error` block with `t("auth.errRateLimited")`.
6. On success: same transition as OAuth (fetch `/api/me`, show avatar chip).

**Email signup:**
1. Toggle to signup mode.
2. Fill display name (optional), email, password.
3. Client-side validation on submit: password < 8 chars → inline error `t("auth.errWeakPassword")`, stop submission.
4. On `EMAIL_IN_USE` from server: inline error below email field.
5. On success: account created, signed in immediately, verification banner appears in lobby.

**Forgot password:**
1. Click "Forgot password?" link → `screen === 'reset-request'`.
2. Enter email → POST `/auth/reset-request`.
3. Server always responds 200 (enumeration-safe). Show `t("auth.resetSent")` success state regardless.
4. "Back to sign in" link returns to `screen === 'auth'`.

### Dropdown Menu

- Opens on single click/tap of `.profile-chip`.
- Closes on: second click on chip, click anywhere outside `.avatar-menu`, Escape key.
- "Sign out": no confirmation — immediate POST `/auth/signout`, optimistic UI (chip vanishes, lobby reverts to guest state; `.error` toast on failure using existing toast pattern).
- "Sign out all devices": clicking replaces dropdown content with 2-button confirmation row (heading + body + confirm/cancel). Dropdown does NOT close — widens slightly to 240px to fit the confirmation. On confirm: POST `/auth/signout-all`, then guest reversion. On cancel: dismiss confirmation, return to normal dropdown state.

### Profile Screen

- Enters via `setScreen('profile')` — lobby content unmounts, profile view mounts with `.rise` animation.
- Profile load: GET `/api/profile/:id` on mount. While loading: skeleton — avatar circle (gray pulse `rgba(255,255,255,0.08)`), name bar (gray pulse), stats row (gray pulse). Uses existing `pulse` keyframe.
- "Back to lobby": `setScreen('lobby')` with `.rise` animation.
- "Resend verification email": POST `/auth/resend-verify`; show `t("auth.verifyLinkSent")` success note inline; button disabled for 60 seconds after click to prevent spam.

### Verification Banner

- Appears in lobby when `authState.emailVerified === false`.
- Dismiss (×): hides for session (sessionStorage key `verifyBannerDismissed`); reappears on next page load.
- "Resend" link: same behavior as profile "Resend" button.

### Password Reset (token flow)

- On page load, if `window.location.search` contains `?token=`, parse token before socket connection. Set `screen === 'reset-confirm'` with token in state.
- Prevents guest from accidentally entering lobby while URL has a reset token active.
- After successful reset: clear token from URL via `history.replaceState`; auto-navigate to `screen === 'auth'` after 2 seconds.

## Accessibility Contracts

- All interactive elements (chip, dropdown items, OAuth buttons, form inputs, submit buttons) are focusable with visible focus rings: `outline: 2px solid var(--gold); outline-offset: 2px`.
- Dropdown menu: `role="menu"` with `role="menuitem"` children.
- Avatar `<img>` has `alt="{displayName}"`. Fallback div has `aria-label="{displayName}"`.
- Sign-out-all confirmation announced via `role="alert"` when it replaces dropdown content.
- Email form: all inputs have `<label>` elements (not just `placeholder`). Labels use existing uppercase `label` style from `style.css`.
- Password inputs: `autocomplete="current-password"` (sign-in) and `autocomplete="new-password"` (signup / reset).
- Email inputs: `autocomplete="email"`.
- Collapsible email section: `aria-expanded` toggled on the trigger button; `aria-controls` pointing to the form panel id.
- Verification banner: `role="status"` so screen readers announce it without interrupting.
- Unverified amber dot on avatar: purely visual (decorative); the profile view and banner carry the accessible copy.
- Reduced-motion: skeleton pulse animation respects `@media (prefers-reduced-motion: reduce)` — swap to `opacity: 0.6` static state (existing pattern).
