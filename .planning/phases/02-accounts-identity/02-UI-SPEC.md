# Phase 2 — UI Design Contract: Accounts & Identity

> Visual and interaction contract for Phase 2 frontend work. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope:** Three surfaces only — (1) home-menu "Sign in with Google" button, (2) signed-in header avatar + dropdown menu, (3) profile view (own + other player, zero-state stats). The in-game battle screen is untouched. Source: CONTEXT.md D-12.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — custom CSS in `public/style.css` |
| Preset | not applicable |
| Component library | none — hand-rolled React components |
| Icon library | Unicode emoji / inline SVG (existing pattern in app.jsx) |
| Font | "Be Vietnam Pro" (body), "Oswald" (headings, labels, buttons, codes) — already loaded via `index.html` implied by `style.css` `font-family` declarations |

**Rule:** Extend `public/style.css` only. Do NOT introduce Tailwind, shadcn, or any new CSS framework. New selectors must follow the existing BEM-lite flat naming pattern (e.g. `.sign-in-btn`, `.avatar-menu`, `.profile-view`).

## Spacing Scale

All values are the existing project 8-point scale as read from `style.css`. Extend; do not override.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, avatar inner padding |
| sm | 8px | Dropdown item vertical padding, avatar chip gap |
| md | 16px | Lobby button vertical padding, profile section spacing |
| lg | 24px | Profile card padding, lobby section gaps |
| xl | 32px | Profile stats row spacing |
| 2xl | 48px | Profile view top margin |
| 3xl | 64px | Not used in this phase |

**Exceptions:**
- Avatar touch target: 44px minimum (height of `.profile-chip` = 34px + 5px top/bottom padding = 44px effective). Source: WCAG 2.5.5 AA minimum touch target.
- Dropdown menu minimum width: 180px (prevents label truncation at narrow viewports).

## Component Inventory

Three net-new UI surfaces for this phase.

### Surface 1: "Sign in with Google" Button (home lobby)

**Location:** Inside `.lobby` panel, below the existing lobby action buttons. Separated by the existing `.divider` element.

**HTML structure:**
```
.lobby
  .divider  (existing pattern — "OR" / "HOẶC")
  button.btn.google-signin  (new class)
    img.google-logo  (Google G logo SVG, 18×18)
    span  (label text)
```

**Visual spec:**
- Width: 100% (matches all other `.btn` elements)
- Height: 50px (padding: 15px top/bottom — matches `.btn` padding spec)
- Background: `rgba(255, 255, 255, 0.92)` — white/off-white to match Google brand identity guidelines; not the gold gradient (gold is for game CTAs, Google has its own brand requirement)
- Text color: `#1f1f1f` (Google brand dark)
- Border: `1px solid rgba(255,255,255,0.20)` at rest; `1px solid rgba(255,255,255,0.50)` on hover
- Border-radius: 12px (matches `.btn`)
- Font: "Be Vietnam Pro", 15px, weight 500 — exception to the Oswald button rule; Google brand requires readable mixed-case, not all-caps Oswald. Label is NOT uppercase.
- Google logo: inline SVG `<svg viewBox="0 0 18 18">` — standard Google G mark, 18×18px, positioned left with 8px gap to label
- Hover: `filter: brightness(1.04)` (subtle lift, consistent with `.btn:hover`)
- Disabled state: `opacity: 0.5; cursor: not-allowed` — shown during OAuth redirect in-flight

**Accessibility:**
- `aria-label="Sign in with Google"` / `aria-label="Đăng nhập bằng Google"` (i18n via `t()`)
- Focus ring: `outline: 2px solid var(--gold); outline-offset: 2px` (matches `.btn:focus-visible`)

### Surface 2: Signed-in Header — Avatar Chip + Dropdown Menu

**Location:** `.topbar-right` (right side of the top bar, replacing the guest state — or added alongside existing buttons when signed in).

**Avatar chip** (`.profile-chip` — already defined in `style.css`):
- Existing styles apply as-is: 28px avatar, 13px name, 20px border-radius, panel background
- Name: truncated at 150px max-width with `text-overflow: ellipsis`
- On mobile ≤380px: name hidden, chip collapses to avatar only (already in `style.css`)
- Cursor: `pointer`; hover: `border-color: var(--gold)` (1px border lift)
- Active (dropdown open): `border-color: var(--gold); box-shadow: 0 0 0 2px rgba(240,193,75,0.25)`

**Dropdown menu** (`.avatar-menu` — new):
- Triggered by: click/tap on `.profile-chip`
- Position: `absolute; top: calc(100% + 6px); right: 0` anchored to the chip
- Width: 200px (min); auto if content wider
- Background: `var(--panel)` with `backdrop-filter: blur(14px) saturate(1.2)` — matches `.lobby`
- Border: `1px solid var(--panel-brd)` = `rgba(120, 180, 255, 0.16)`
- Border-radius: 12px
- Box-shadow: `0 12px 36px rgba(2,12,30,0.55)`
- z-index: 80 (above `.overlay` at 50, below `.notice-toast` at 200)
- Animation: `rise 0.25s cubic-bezier(.34,1.4,.4,1)` (matches existing `.lobby` animation)
- Close on: click outside, Escape key, item selection

**Dropdown items** (`.avatar-menu-item` — new):
- Padding: 11px 16px
- Font: "Be Vietnam Pro", 14px, weight 400, color `#dbe7f3`
- Hover background: `rgba(255,255,255,0.06)`
- Hover left indicator: `border-left: 2px solid var(--gold)`; padding-left compensates by 2px
- Icon: 16px emoji prefix with 8px gap (matches existing power-btn icon pattern)
- Separator: `border-top: 1px solid var(--panel-brd)` before destructive item group

**Dropdown item list (ordered):**
1. "View profile" / "Xem hồ sơ" — navigates to profile screen state
2. "Sign out" / "Đăng xuất" — POST `/auth/signout`; returns to lobby as guest
3. --- separator ---
4. "Sign out all devices" / "Đăng xuất tất cả thiết bị" — color `var(--hit)` = `#ff4d5e`; triggers inline confirmation (see Copywriting)

### Surface 3: Profile View (screen state / overlay)

**Trigger:** Selecting "View profile" from dropdown, or any future `<a>` linking to a profile by opaque user ID.

**Implementation pattern:** New screen state in the existing `useState(screen)` SPA — not a route, not a modal overlay. Renders in place of `.lobby` when `screen === 'profile'`. Source: CONTEXT.md D-12, code_context "Screen-state-driven SPA."

**Layout — `.profile-view` (new):**
- Max-width: 480px (matches `.app` and `.lobby`)
- Margin: `6vh auto 0` (matches `.lobby`)
- Background/border/backdrop: same as `.lobby` — glass panel
- Padding: 34px 30px (matches `.lobby`)
- Animation: `rise 0.55s cubic-bezier(.34,1.4,.4,1) both` (matches `.lobby`)

**Profile header section (`.profile-header`):**
- Avatar: 64px circle; `object-fit: cover`; fallback: `var(--sky)` background + 28px Oswald initial letter
- Display name: 20px Oswald, weight 700, `#eaf2ff`; max-width 260px, truncated
- Member since: 12px, `#a9ccec`, formatted as "Member since {month} {year}" / "Thành viên từ tháng {m} năm {y}"
- Layout: row — avatar left (64px), name + since stacked right; gap 16px; align-items center

**Stats row (`.profile-stats`) — zero-state scaffold:**
- 3-column grid (wins | separator | losses); equal width
- Each stat cell: label on top (12px uppercase `#a9ccec`), figure below (22px Oswald bold `var(--gold)`)
- Zero-state figure: "0" — no special treatment; same style as real numbers (D-10: no UI rework when Phase 3 fills real data)
- "No games yet" sub-line: 13px `#a9ccec` centered below the stats row; only shown when all 3 figures are 0

**Back navigation:**
- Single `button.btn.ghost` labeled "Back" / "Quay lại" at bottom of the profile view
- Width: auto; padding: 10px 20px; not full-width

**Own vs. other-player profile:**
- Own profile: shows "Sign out" ghost button below stats (shortcut — same as dropdown item)
- Other player profile: no sign-out; adds "Challenge" button placeholder (disabled, `opacity: 0.4`) for Phase 5 wiring — renders it structurally so Phase 5 requires no layout change

## Interaction Contracts

### Sign-in Flow

1. User taps "Sign in with Google" button.
2. Button enters disabled state immediately (`opacity: 0.5; cursor: not-allowed`) — prevents double-tap.
3. Browser navigates to `/auth/google` (full-page redirect — no popup, no iframe).
4. On return: if `?authError=1` in URL, show inline `.error` block in the lobby with `t("auth.errFailed")`. Button re-enables.
5. On success: React reads `/api/me` (or session-hydrated initial state from server-rendered props) and transitions to signed-in state. Avatar chip appears in `.topbar-right`.

### Dropdown Menu

- Opens on single click/tap of `.profile-chip`.
- Closes on: second click on chip, click anywhere outside `.avatar-menu`, Escape key.
- "Sign out": no confirmation — immediate POST `/auth/signout` on click, optimistic UI (chip vanishes, lobby reverts to guest state; error toast on failure).
- "Sign out all devices": clicking opens an inline confirmation within the dropdown (not a full modal). The dropdown widens slightly to accommodate the 2-button confirmation row. On confirm: POST `/auth/signout-all`, then same guest reversion.

### Profile Screen

- Enters via `setScreen('profile')` — lobby content unmounts, profile view mounts with `.rise` animation.
- "Back" button: `setScreen('lobby')` — same `rise` animation.
- Profile load: GET `/api/profile/:id` on mount. While loading: show skeleton — avatar circle (gray pulse), name bar (gray pulse), stats row (gray pulse). Skeleton uses `animation: pulse 1.6s ease-in-out infinite` (existing keyframe).
- Profile 404 (viewing another player who doesn't exist): show `.error` block with "Player not found." / "Không tìm thấy người chơi."

## Accessibility Contracts

- All interactive elements (chip, dropdown items, sign-in button) are focusable and have visible focus rings matching the existing `outline: 2px solid var(--gold); outline-offset: 2px` pattern.
- Dropdown menu uses `role="menu"` with `role="menuitem"` children.
- Avatar `<img>` has `alt="{displayName}"`. Fallback div has `aria-label="{displayName}"`.
- Sign-out confirmation is announced via `role="alert"` when it appears inside the dropdown.
- Reduced-motion: skeleton pulse animation respects `@media (prefers-reduced-motion: reduce)` — swap to `opacity: 0.6` static state.
