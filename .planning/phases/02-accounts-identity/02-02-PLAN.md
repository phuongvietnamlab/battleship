<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> /auth/google callback | cross-site top-level redirect from Google; carries the state nonce in the session cookie |
| client -> Socket.IO handshake | session cookie crosses into the websocket upgrade; userId derived server-side only |
| GoogleStrategy verify callback -> DB | sub/name from Google token; name sanitized in Plan 01's linkOrPromoteAccount |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Spoofing | OAuth callback (CSRF / state forgery) | mitigate | GoogleStrategy state:true generates + validates a per-flow nonce; forged/missing state rejected (SEC-05) |
| T-02-06 | Elevation of Privilege | session fixation post-login | mitigate | Passport 0.6+ calls req.session.regenerate() automatically on req.logIn (SEC-05) |
| T-02-07 | Spoofing | session hijacking via cookie | mitigate | cookie httpOnly:true, secure:'auto' (+ trust proxy 1), sameSite:'lax' (lax required so OAuth callback cookie is sent — Pitfall 5) |
| T-02-08 | Information Disclosure | Google access_token leak | mitigate | verify callback never stores accessToken; serializeUser writes only users.id |
| T-02-09 | Denial of Service | auth-route brute force | mitigate | authRateLimit (RateLimiterMemory 10/60s per IP) on /auth/google + callback |
| T-02-10 | Spoofing | two sessionMiddleware instances -> socket auth never matches | mitigate | single const sessionMiddleware shared by app.use and io.engine.use (Pitfall 2) |
| T-02-11 | Tampering | clientId smuggled via query into link | mitigate | pendingClientId persisted in server-side session (not trusted from callback URL); link SQL parameterized |
</threat_model>
