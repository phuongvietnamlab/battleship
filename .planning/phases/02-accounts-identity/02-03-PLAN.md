<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> POST /auth/signout-all | revocation request; must be tied to the authenticated req.user, never a client-supplied user_id |
| session row -> user_id column | the DELETE key; written server-side at login only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-12 | Elevation of Privilege | sign-out-all targeting another user | mitigate | userId taken from req.user.id only (401 if absent); never from request body — client cannot revoke others |
| T-02-13 | Elevation of Privilege | stale session after sign-out-all (race) | mitigate | single indexed DELETE FROM session WHERE user_id=$1 removes all rows atomically; subsequent requests find no session |
| T-02-14 | Tampering | sign-out-all SQL injection via user_id | mitigate | parameterized $1 binding; user_id is an integer from req.user |
| T-02-15 | Spoofing | sign-out fails silently (sync logout) | mitigate | req.logout(callback) — Passport 0.6+ async; session.destroy only after logout resolves (Pitfall 3) |
</threat_model>
