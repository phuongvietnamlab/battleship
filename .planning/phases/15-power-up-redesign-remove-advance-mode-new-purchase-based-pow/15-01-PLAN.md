# Plan 15-01: Legacy Removal — Remove Advance Mode & Old Power-up System

## Objective
Remove all advance-mode code paths, old power-up spawning, mid-match shop, and mode selection UI. After this plan, the game has a single unified mode with no power-ups (power-ups are added back in later plans).

## Tasks

### Task 1: Remove server-side advance mode logic

**Files:** `server.js`

1. Delete `maybeSpawn()` function and its call site in `doShot` (line ~1420)
2. Delete `powerupsForAttacker()` function
3. Delete `POWERS` array, old `newInv()` function
4. Delete `buyPowerup` socket handler (lines 2223–2297) and `buyPowerupLimiter`
5. Delete old constants: `POWERUP_FIXED_FLOOR`, `POWERUP_STAKE_PCT`, `POWERUP_CAP_PER_MATCH` (lines 155-160)
6. Remove mode parameter from `createRoom`: line ~1653 `const mode = ...` → always omit or set to null
7. Remove mode from `joinQueue` entry creation: line ~1756
8. Remove mode from queue pairing logic (`findPair`): lines ~1504, ~1515
9. Remove `room.powerups = {}` from room creation (line ~1592) and rematch reset (line ~1995)
10. Remove power-up board state from `doShot`: lines ~1370-1393 (pmap pickup logic)
11. Remove `oppBoughtPowerup` emit (line ~2289)
12. In `syncPayload`: remove `powerups`, `purchasesRemaining`, `powerupPrice` fields; keep `mode: "classic"` for backwards compat
13. In `roomPublic()`: remove mode or hardcode "classic"
14. Keep `room.mode` in serialize/restore for existing rooms during migration
15. Keep `recordMatch(..., room.mode, ...)` — always passes "classic" for new games
