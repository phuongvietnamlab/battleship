# Plan 15-04: Client UI — Placement Shop + Decoy Placement

## Objective
Build the power-up shop on the placement screen. Players see a horizontal row of 4 power-up options, can purchase up to 2, and if Decoy is purchased, must place it on the grid before pressing Ready.

## Tasks

### Task 1: New PlacementShop component

**Files:** `public/app.jsx`

Create a `PlacementShop` component:

```jsx
function PlacementShop({ stake, balance, inventory, purchaseCount, onBuy, disabled }) {
  const price = Math.round(stake * 0.10);
  const maxReached = purchaseCount >= 2;
  const canAfford = balance >= price;
  
  return (
    <div className="placement-shop">
      <div className="shop-header">⚡ Power-ups · {price} 💰 each</div>
      <div className="shop-row">
        {[
          { type: "sonar", icon: "🔊", name: t("pw.sonar") },
          { type: "cross", icon: "➕", name: t("pw.cross") },
          { type: "decoy", icon: "🪤", name: t("pw.decoy") },
          { type: "scatter", icon: "🌠", name: t("pw.scatter") },
        ].map(({ type, icon, name }) => (
          <button key={type} className="shop-item"
            onClick={() => onBuy(type)}
            disabled={disabled || maxReached || !canAfford || (type === "decoy" && inventory.decoy >= 1)}>
            <span className="shop-icon">{icon}</span>
            <span className="shop-name">{name}</span>
          </button>
        ))}
      </div>
      {maxReached && <div className="shop-cap">{t("shop.capReached")}</div>}
      {!maxReached && !canAfford && <div className="shop-balance">{t("wallet.insufficientBalance")}</div>}
    </div>
  );
}
```

### Task 3: Implement purchase flow (client-side)

**Files:** `public/app.jsx`

In App component:
1. Add state: `const [placementInv, setPlacementInv] = useState(newInv())`
2. Add state: `const [placementPurchases, setPlacementPurchases] = useState(0)`
3. Function `handlePlacementBuy(type)`:
   ```javascript
   function handlePlacementBuy(type) {
     socket.emit("buyPlacementPowerup", { type }, (res) => {
       if (res && res.ok) {
         setPlacementInv(prev => ({ ...prev, [type]: (prev[type] || 0) + 1 }));
         setPlacementPurchases(n => n + 1);
         // Balance updates via balanceUpdate event
         if (type === "decoy") setDecoyPending(true);
       } else {
         showNotice(t("err." + (res?.code || "UNKNOWN")));
       }
     });
   }
   ```

### Task 4: Implement Decoy placement mode

**Files:** `public/app.jsx`

1. Add state: `const [decoyPending, setDecoyPending] = useState(false)`
2. Add state: `const [decoyCell, setDecoyCell] = useState(null)`
3. When `decoyPending === true`:
   - Ready button disabled
   - Grid cells get `.placeable` class for empty (non-ship) cells
   - Show hint: "Tap an empty cell to place your decoy"
   - On grid cell click (own board, empty cell): `setDecoyCell({r, c})`, `setDecoyPending(false)`
   - On grid cell click (ship cell): show error toast "Cannot place decoy on a ship"
4. Decoy marker on own board:
   - If `decoyCell` is set, render a 🪤 icon on that cell
   - CSS class `.decoy-marker` with subtle styling
5. If player rearranges ships (drag/random) and ship now overlaps decoy:
   - Reset `decoyCell = null`, `decoyPending = true`
   - Show hint: "Decoy position invalidated — place it again"
