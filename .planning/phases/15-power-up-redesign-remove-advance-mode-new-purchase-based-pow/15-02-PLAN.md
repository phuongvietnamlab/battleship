# Plan 15-02: New Purchase System — Server-side Placement-Phase Power-up Shop

## Objective
Implement the server-side infrastructure for purchasing power-ups during the placement phase. Players can buy up to 2 power-ups before battle starts. Decoy placement is validated with ship placement.

## Tasks

### Task 1: Define new power-up constants and inventory

**Files:** `server.js`

1. Add new constants:
   ```javascript
   const NEW_POWERS = ["sonar", "cross", "decoy", "scatter"];
   const POWERUP_MAX_PER_MATCH = 2;
   const POWERUP_PRICE_PCT = 0.10;
   ```
2. New `newInv()` function:
   ```javascript
   function newInv() { return { sonar: 0, cross: 0, decoy: 0, scatter: 0 }; }
   ```
3. Add `room.purchases = {}` to room creation
4. Add `room.players[clientId].decoyCell = null` to player init

### Task 2: Implement `buyPlacementPowerup` socket handler

**Files:** `server.js`

New socket event handler with validations:
1. Guard: room exists, player is in room, game not started
2. Guard: `room.stake > 0` (free matches cannot buy)
3. Guard: player has `userId` (authenticated, not guest)
4. Guard: valid type (must be in `NEW_POWERS`)
5. Guard: `room.purchases[clientId] < POWERUP_MAX_PER_MATCH` (max 2)
6. Guard: if `type === "decoy"`, `me.inv.decoy < 1` (max 1 decoy)
7. Calculate price: `Math.round(room.stake * POWERUP_PRICE_PCT)`
8. Rate limit: reuse `buyPowerupLimiter` (3/60s)
9. Call `debitWallet(userId, price, "powerup_purchase", referenceId)`
10. On success: `me.inv[type]++`, `room.purchases[clientId]++`
11. Emit `balanceUpdate` to buyer only
12. Do NOT emit anything to opponent (secrecy)
13. Return `cb({ ok: true, type, price, newBalance, remaining })`

### Task 3: Extend `placeShips` to accept decoy cell

**Files:** `server.js`

1. Modify `placeShips` handler to accept new payload shape:
   ```javascript
   // Old: socket.on("placeShips", (ships, cb) => ...)
   // New: socket.on("placeShips", (arg, cb) => ...)
   // arg = ships (array, backwards compat) OR { ships, decoyCell }
   ```
2. Backwards-compatible parsing:
   ```javascript
   const ships = Array.isArray(arg) ? arg : arg.ships;
   const decoyCell = Array.isArray(arg) ? null : (arg.decoyCell || null);
   ```
3. If player has `me.inv.decoy > 0` and `decoyCell` is null → reject with `DECOY_NOT_PLACED`
4. If `decoyCell` provided:
   - Validate `inBounds(decoyCell.r, decoyCell.c)`
   - Validate cell not in `occ` (not on a ship)
   - Add decoy key to `occ`: `occ.add(decoyCell.r + "," + decoyCell.c)`
   - Store: `room.players[clientId].decoyCell = decoyCell.r + "," + decoyCell.c`
5. Existing `validatePlacement()` runs on ships as before

### Task 4: Update syncPayload for new system

**Files:** `server.js`

1. Include `inv` (new shape) in sync payload
2. Include `decoyCell` for the player's own view (so they can see their decoy marker)
3. Include `stake` (already there)
4. Include `purchasesRemaining: POWERUP_MAX_PER_MATCH - (room.purchases[clientId] || 0)`
5. Include `powerupPrice: room.stake > 0 ? Math.round(room.stake * POWERUP_PRICE_PCT) : 0`
6. Do NOT include opponent's inventory in sync (secrecy)

### Task 5: Update rematch/round reset

**Files:** `server.js`

1. On rematch: reset `room.purchases = {}`
2. Reset each player's `inv = newInv()`, `decoyCell = null`
3. On new round (Bo3): same resets

## Success Criteria
- [ ] `buyPlacementPowerup` event works: debit wallet, add to inventory
- [ ] Max 2 purchase cap enforced
- [ ] Max 1 decoy cap enforced
- [ ] Free matches (stake=0) reject purchases
- [ ] Guest players (no userId) reject purchases
- [ ] `placeShips` accepts and validates decoy cell
- [ ] Decoy cell added to `occ` but not `ships`
- [ ] Opponent receives zero information about purchases
- [ ] syncPayload includes new inventory shape
- [ ] Rematch properly resets purchase state
