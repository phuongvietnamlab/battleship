# Plan 15-06: Integration Testing + Edge Cases

## Objective
Verify the complete power-up system works end-to-end. Test edge cases, reconnection, and ensure no regressions in core gameplay.

## Tasks

### Task 1: Manual integration test — full game flow

1. Create a wagered match (stake > 0) between 2 authenticated players
2. Player A buys Sonar + Cross Missile during placement
3. Player B buys Decoy + Scatter during placement
4. Player B places decoy on empty cell
5. Both press Ready → game starts
6. Verify: Player A's PowerBar shows Sonar + Cross
7. Verify: Player B's PowerBar shows Scatter only (Decoy already placed)
8. Player A uses Sonar on a row containing Player B's ship → "YES"
9. Verify: turn passes to Player B
10. Player B uses Scatter → 2-3 random cells fire → turn passes to Player A
11. Player A fires normally at decoy cell → "HIT!" → Player A keeps turn (normal shot)
12. Player A fires around decoy → all miss → realizes it's fake
13. Player A uses Cross Missile on suspected ship area → 5 cells fire → turn passes to Player B
14. Continue until one player wins

### Task 2: Edge case — Decoy doesn't count for win

1. Set up game where all 5 real ships are sunk but decoy hasn't been hit
2. Verify game ends with correct winner
3. Verify game ends even though "HIT" cell exists that isn't part of any sunk ship announcement

### Task 3: Edge case — Sonar + Decoy interaction

1. Place decoy on row 3
2. No real ships on row 3
3. Opponent uses Sonar on row 3
4. Expected: "YES" (decoy cell is in `occ`)
5. This is intentional — decoy fools sonar

### Task 4: Edge case — Cross Missile boundary cells

1. Use Cross Missile on corner cell (0,0): only 3 cells fire (center + right + down)
2. Use Cross Missile on edge cell (0,5): only 4 cells fire (center + left + right + down)
3. Use Cross Missile targeting cells already shot: those cells skipped

### Task 5: Edge case — Scatter Blast with few remaining cells

1. Late game: only 1 unshot cell remains
2. Activate Scatter Blast → fires at that 1 cell
3. Late game: 0 unshot cells → Scatter rejected with NO_CELLS error

### Task 6: Edge case — Purchase limits

1. Buy 2 power-ups → try buying 3rd → rejected with PURCHASE_CAP_REACHED
2. Buy 1 decoy → try buying 2nd decoy → rejected (max 1 decoy)
3. Balance = 3, price = 5 → rejected with INSUFFICIENT_BALANCE
4. Free match (stake=0) → shop not visible, server rejects if attempted

### Task 7: Edge case — Reconnection

1. Player purchases 2 power-ups, places decoy, presses Ready
2. Player disconnects and reconnects within grace period
3. Verify: syncPayload includes correct inventory, decoyCell
4. Verify: PowerBar shows correct power-ups on reconnect
5. Verify: enemy board shows decoy cell as "HIT" if it was already hit

### Task 8: Edge case — Guest and bot games

1. Guest player joins wagered match → shop not visible (no wallet)
2. Bot game → shop not visible (no stake, vsBot)
3. Bot play works normally — no crashes from removed advance mode code

### Task 9: Edge case — Rematch reset

1. Complete a game with power-ups
2. Rematch → verify: inventory reset, purchases reset, decoyCell reset
3. Shop available again for new round of purchases

### Task 10: Verify match history compatibility

1. Load match history page
2. Verify old advance-mode games display correctly (mode column still "advance")
3. Verify new games record with mode = "classic"
4. No errors in history display

## Success Criteria
- [ ] Full game flow works end-to-end
- [ ] Decoy: HIT reported, never triggers sunk, game ends when 5 real ships sunk
- [ ] Sonar: detects decoy (intentional), correct YES/NO on real ships
- [ ] Cross Missile: handles boundary cells, skips already-shot
- [ ] Scatter Blast: handles few remaining cells, rejects on 0
- [ ] Purchase limits enforced (2 total, 1 decoy)
- [ ] Reconnection preserves all power-up state
- [ ] Guest/bot games unaffected
- [ ] Rematch properly resets
- [ ] Match history displays old + new games
